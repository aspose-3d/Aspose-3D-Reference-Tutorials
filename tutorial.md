# Building a No-Code Agent with OpenCode

I want to share how I built a fully automated agent that handles pull request merging for the Aspose.3D Reference Tutorials repository—without writing a single line of code.

Here's the story: Muhammad Muqarrab developed an agent that periodically analyzes tutorial pages and creates pull requests with updated content. At first, I was merging these PRs manually, which meant carefully reviewing each markdown file and fixing any incorrect code snippets (LLMs love to hallucinate API names).

It was tedious work. Then I discovered OpenCode, and everything changed.

Now I have a completely automated system that:
- Reviews every PR
- Verifies all API calls are correct
- Fixes any issues automatically
- Merges, pushes, and closes the PRs

Let me show you how I built it.

## Setting Up the MCP Server

The secret sauce is an MCP (Model Context Protocol) server that can verify Aspose.3D APIs and check code for errors. OpenCode needs this server to understand what's valid and what's not.

First, grab the MCP server code:

```bash
git clone git@gitlab.recruitize.ai:wuhan-3d/ai-agents.git
```

The server needs API documentation to work its magic. You'll need two JSON files:
- `Aspose.3D.Java.json` - Generated by the build tool
- `Aspose.3D.NET.json` - Generated by the porting tool

These files contain all the API details that the MCP server needs to verify code snippets.

Once you have the files, start the server:

```bash
python assist-server.py
```

I run mine behind nginx at `dev.acrody.site`, so it's accessible via HTTPS.

## Connecting OpenCode to the MCP Server

Now tell OpenCode about your new MCP server. Add this to your OpenCode config:

```json
{
  "mcp": {
    "3d-assistant": {
      "type": "remote",
      "url": "https://dev.acrody.site/mcp",
      "enabled": true
    }
  }
}
```

With this connection, OpenCode gains two superpowers:
- **`ask_api`** - Can look up any API to see if it exists
- **`verify_code`** - Can check code snippets for errors

This is what makes the whole thing possible—OpenCode can now verify code snippets on its own, without needing me to double-check everything.

## Giving OpenCode Instructions

The next step is simple: create an `AGENTS.md` file in your repository and tell OpenCode what you want it to do. Here's what mine says:

```
This repository contains tutorials written in markdown, you will merge all pull requests.

In the pull requests there's only markdown files, which may contain code snippet inside, 
you will use tool to verify if the code snippet contains invalid API calls then you should 
correct the code snippets.
```

That's it. OpenCode now knows the mission: merge PRs, verify code, fix anything that's wrong.

## Using the Agent

Here's the beauty of it—to run the agent, I just tell OpenCode:

```
proceed
```

That's literally all I type. OpenCode reads `AGENTS.md`, understands what needs to happen, and goes to work.

If you want it to run automatically (which I definitely recommend), add this to your crontab or webhook:

```bash
opencode run 'proceed'
```

Now the agent runs whenever you want it to, completely hands-free.

## What the Agent Does

When I say "proceed," OpenCode processes each open PR through this entire workflow:

1. It checks GitHub for any open pull requests
2. Checks out the branch for each PR
3. Reads through all the markdown files
4. Finds every code snippet and verifies the APIs
5. Runs the code through the verifier to catch any errors
6. If there are issues, fixes them automatically
7. Verifies the fixed code one more time
8. Merges the branch into main
9. Pushes everything to the repository
10. Closes the PR on GitHub

I used to do all of this manually for every single PR. Now I just type "proceed" and watch it happen.

It's actually kind of satisfying to watch.

## The Real Value

What I love about this approach is that I didn't have to write any custom code. No Python scripts, no GitHub Actions workflows, no API integrations. Just a simple instruction file and a pre-built MCP server.

OpenCode combines everything it needs:
- It reads my instructions from `AGENTS.md`
- It uses the MCP tools to verify APIs and check code
- It knows how to use git and GitHub commands
- It understands how to fix code when it finds problems

The result: a reliable automated agent that keeps our tutorial documentation accurate without requiring constant human oversight.

## Wrapping Up

To build your own agent like this:

1. Set up an MCP server with the API documentation you need
2. Connect OpenCode to that server
3. Write clear instructions in `AGENTS.md`
4. Say "proceed"

That's it. A fully functional agent, zero custom code.
