# 3D फ़ाइल आकार घटाएँ – Aspose.3D for Java के साथ दृश्यों को संकुचित करें

## परिचय

यदि आप वेब पर, ईमेल के माध्यम से, या क्लाउड बकेट में 3D एसेट्स वितरित कर रहे हैं, तो बड़ी फ़ाइल आकार जल्दी ही बाधा बन सकते हैं। इस ट्यूटोरियल में आप **3D फ़ाइल आकार को कैसे घटाएँ** सीखेंगे, Aspose.3D for Java का उपयोग करके 3D दृश्यों को संकुचित करके। हम एक सीन बनाने, ऑब्जेक्ट जोड़ने, ट्रांसफ़ॉर्मेशन को समायोजित करने, और अंत में सीन को संपीड़न विकल्पों के साथ सहेजने की प्रक्रिया को देखेंगे, जिससे दृश्य गुणवत्ता बनी रहे जबकि फ़ाइल काफी घटे।

## त्वरित उत्तर
- **“reduce 3d file size” का क्या अर्थ है?** इसका मतलब है 3‑D फ़ाइल पर संपीड़न तकनीकों को लागू करना ताकि उसका डिस्क पर आकार छोटा हो जाए बिना ज्यामिति या टेक्सचर की सटीकता खोए।  
- **Aspose.3D में कौन सा फ़ॉर्मेट संपीड़न का समर्थन करता है?** AMF (Additive Manufacturing File) फ़ॉर्मेट, `AmfSaveOptions` का उपयोग करके।  
- **क्या संपीड़न के लिए लाइसेंस चाहिए?** विकास के लिए ट्रायल काम करता है; उत्पादन के लिए व्यावसायिक लाइसेंस आवश्यक है।  
- **क्या संपीड़न लॉसलेस है?** हाँ, Aspose.3D का अंतर्निहित संपीड़न ज्यामिति और टेक्सचर के लिए लॉसलेस है।  
- **मैं कितना आकार घटाव की उम्मीद कर सकता हूँ?** आमतौर पर 30‑60 % तक, सीन की जटिलता और टेक्सचर संख्या पर निर्भर करता है।

## Aspose.3D में सीन संपीड़न क्या है?
सीन संपीड़न 3‑D सीन को अधिक संक्षिप्त प्रतिनिधित्व में एन्कोड करने की प्रक्रिया है। Aspose.3D AMF फ़ॉर्मेट की अंतर्निहित gzip‑शैली की संपीड़न का उपयोग करता है, जिससे आप बड़े मॉडल को अधिक कुशलता से भेज सकते हैं।

## 3D फ़ाइल आकार घटाने के कारण
- **तेज़ डाउनलोड** – सीमित बैंडविड्थ वाले उपयोगकर्ताओं को सुगम अनुभव मिलता है।  
- **कम स्टोरेज लागत** – क्लाउड स्टोरेज शुल्क प्रति GB गणना किया जाता है।  
- **बेहतर प्रदर्शन** – छोटे फ़ाइलें ब्राउज़र और गेम इंजन में तेज़ लोड होती हैं।  
- **आसान शेयरिंग** – ईमेल अटैचमेंट और सहयोग प्लेटफ़ॉर्म अक्सर आकार सीमा रखते हैं।

## पूर्वापेक्षाएँ
शुरू करने से पहले, सुनिश्चित करें कि आपके पास है:

- Java Development Kit (JDK) 8 या उससे नया स्थापित हो।  
- Aspose.3D for Java लाइब्रेरी आधिकारिक साइट से डाउनलोड की गई – आप डाउनलोड लिंक [यहाँ](https://releases.aspose.com/3d/java/) पा सकते हैं।  
- एक Java IDE (IntelliJ IDEA, Eclipse, या VS Code) ताकि आप नमूना प्रोजेक्ट बना और चला सकें।

## पैकेज आयात करें
अपने Java स्रोत फ़ाइल में आवश्यक Aspose.3D क्लासेज़ जोड़ें:

```java
import com.aspose.threed.AmfSaveOptions;
import com.aspose.threed.Box;
import com.aspose.threed.Scene;
import com.aspose.threed.Transform;
import com.aspose.threed.Vector3;
```

## स्टेप‑बाय‑स्टेप गाइड

### स्टेप 1: अपना Java प्रोजेक्ट सेट अप करें
अपने पसंदीदा IDE में एक नया Java प्रोजेक्ट बनाएं और Aspose.3D JAR फ़ाइलों को प्रोजेक्ट के क्लासपाथ में जोड़ें। इससे कंपाइलर आयातित क्लासेज़ को ढूँढ़ सकेगा।

### स्टेप 2: एक नया 3D सीन इनिशियलाइज़ करें
सबसे पहले एक खाली सीन ऑब्जेक्ट बनाकर शुरू करें। `Scene` क्लास सभी ज्यामिति, लाइट्स, कैमरा, और पदानुक्रम के लिए कंटेनर है।

```java
// The path to the documents directory.
String MyDir = "Your Document Directory";

Scene scene = new Scene();
```

### स्टेप 3: एक सरल बॉक्स ज्यामिति जोड़ें
प्रदर्शन के लिए, हम सीन में एक बॉक्स प्रिमिटिव जोड़ेंगे। `Box` क्लास एक क्यूब बनाता है जिसे हम ट्रांसफ़ॉर्म कर सकते हैं।

```java
Box box = new Box();
Transform tr = scene.getRootNode().createChildNode(box).getTransform();
tr.setScale(new Vector3(12, 12, 12));
tr.setTranslation(new Vector3(10, 0, 0));
```

### स्टेप 4: बॉक्स को कस्टमाइज़ करें (स्केल, रोटेशन, पोज़िशन)
आप उसी बॉक्स को संशोधित कर सकते हैं या अतिरिक्त इंस्टेंस जोड़ सकते हैं। नीचे हम स्केल बदलते हैं और ऑब्जेक्ट को घुमाने के लिए यूलेर एंगल लागू करते हैं।

```java
tr = scene.getRootNode().createChildNode(box).getTransform();
tr.setScale(new Vector3(5, 5, 5));
tr.setEulerAngles(new Vector3(50, 10, 0));
```

### स्टेप 5: संपीड़न सक्षम करके सीन सहेजें
**3d फ़ाइल आकार घटाने** की कुंजी `AmfSaveOptions` में है। `setEnableCompression(true)` सेट करके AMF फ़ाइल के अंदर gzip संपीड़न सक्रिय करें।

```java
AmfSaveOptions opt = new AmfSaveOptions();
opt.setEnableCompression(true);   // Turn on compression to shrink file size
scene.save(MyDir + "compressed_scene.amf", opt);
```

> **प्रो टिप:** यदि आपको डिबगिंग के लिए मूल अनकम्प्रेस्ड संस्करण रखना है, तो `setEnableCompression(false)` के साथ एक दूसरी कॉपी सहेजें।

ऊपर दिए गए चरणों को किसी भी अतिरिक्त ऑब्जेक्ट के लिए दोहराएँ जिसे आप सीन में शामिल करना चाहते हैं। प्रत्येक ऑब्जेक्ट उसी संकुचित कंटेनर में संग्रहीत होगा, जिससे कुल फ़ाइल आकार कम रहेगा।

## सामान्य समस्याएँ और समाधान
| समस्या | कारण | समाधान |
|-------|-------|-----|
| **सहेजी गई फ़ाइल अभी भी बड़ी है** | संपीड़न निष्क्रिय है या ऐसा फ़ॉर्मेट उपयोग किया गया है जो इसका समर्थन नहीं करता (जैसे, OBJ). | सुनिश्चित करें कि `opt.setEnableCompression(true)` है और **AMF** के रूप में सहेँ। |
| **लोड के बाद टेक्सचर नहीं दिख रहे** | टेक्सचर एम्बेड नहीं किए गए थे; पाथ बाहरी है। | `scene.getRootNode().getMaterial().setTexture(...).setEmbed(true)` का उपयोग करें। |
| **बड़ी सीन पर OutOfMemoryError** | सहेजने से पहले पूरी सीन को मेमोरी में लोड किया जा रहा है। | `AmfSaveOptions.setEnableStreaming(true)` के माध्यम से स्ट्रीमिंग मोड सक्षम करें। |
