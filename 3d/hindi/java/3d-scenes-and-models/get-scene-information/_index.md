# Java में FBX कैसे सहेजें और 3D सीन जानकारी प्राप्त करें

## परिचय

यदि आप **FBX फ़ाइल को सहेजने** के साथ-साथ अपने 3D सीन से उपयोगी मेटाडेटा निकालने के लिए एक स्पष्ट, व्यावहारिक गाइड की तलाश में हैं, तो आप सही जगह पर आए हैं। इस ट्यूटोरियल में हम **Aspose.3D Java** लाइब्रेरी का उपयोग करके हर चरण को समझेंगे: सीन बनाना, **एप्लिकेशन नाम सेट करना**, **मापन इकाइयों को परिभाषित करना**, और अंत में **सीन को FBX में एक्सपोर्ट करना**। अंत तक आपके पास एक तैयार‑to‑use FBX फ़ाइल होगी जिसमें वह एसेट जानकारी होगी जिसकी आपको डाउनस्ट्रीम पाइपलाइनों के लिए आवश्यकता है।

## त्वरित उत्तर
- **मुख्य लक्ष्य क्या है?** एक FBX फ़ाइल सहेजना जिसमें कस्टम एसेट जानकारी हो।  
- **कौन सी लाइब्रेरी उपयोग की गई?** Aspose.3D for Java।  
- **क्या लाइसेंस की जरूरत है?** विकास के लिए फ्री ट्रायल चलती है; प्रोडक्शन के लिए कमर्शियल लाइसेंस आवश्यक है।  
- **क्या मापन इकाइयाँ बदल सकते हैं?** हाँ – `setUnitName` और `setUnitScaleFactor` का उपयोग करें।  
- **आउटपुट कहाँ सहेजा जाता है?** `scene.save(...)` में आप जो पाथ देंगे, उसी में।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास:

- कोर Java सिंटैक्स की ठोस समझ हो।  
- **Aspose.3D for Java** डाउनलोड किया हुआ और अपने प्रोजेक्ट में जोड़ा हुआ हो (आप इसे आधिकारिक [Aspose 3D download page](https://releases.aspose.com/3d/java/) से प्राप्त कर सकते हैं)।  
- आपका पसंदीदा Java IDE (IntelliJ IDEA, Eclipse, NetBeans, आदि) सही तरीके से कॉन्फ़िगर किया हुआ हो।

## पैकेज इम्पोर्ट करें

अपने Java स्रोत फ़ाइल में, Aspose.3D क्लासेज़ को इम्पोर्ट करें जो सीन हैंडलिंग और फ़ाइल‑फ़ॉर्मेट सपोर्ट प्रदान करते हैं।

```java
import com.aspose.threed.FileFormat;
import com.aspose.threed.Scene;
```

> **प्रो टिप:** अनावश्यक डिपेंडेंसीज़ से बचने और कंपाइल टाइम को तेज़ करने के लिए इम्पोर्ट सूची को न्यूनतम रखें।

## FBX फ़ाइल सहेजने की प्रक्रिया क्या है?

नीचे एक संक्षिप्त, चरण‑दर‑चरण walkthrough दिया गया है जो **एसेट जानकारी को सीन में जोड़ना** और फिर **सीन को FBX में एक्सपोर्ट करना** दिखाता है।

### चरण 1: 3D सीन को इनिशियलाइज़ करें

सबसे पहले, एक खाली `Scene` ऑब्जेक्ट बनाएं। यह सभी जियोमेट्री, लाइट्स, कैमरा और एसेट मेटाडेटा का कंटेनर होगा।

```java
// ExStart:AddAssetInformationToScene
Scene scene = new Scene();
```

### चरण 2: एप्लिकेशन और वेंडर जानकारी सेट करें

कस्टम मेटाडेटा जोड़ने से डाउनस्ट्रीम टूल्स फ़ाइल के स्रोत की पहचान कर सकते हैं। यहाँ हम `AssetInfo` ऑब्जेक्ट का उपयोग करके **एप्लिकेशन नाम** और वेंडर सेट करते हैं।

```java
scene.getAssetInfo().setApplicationName("Egypt");
scene.getAssetInfo().setApplicationVendor("Manualdesk");
```

> **यह क्यों महत्वपूर्ण है:** कई पाइपलाइन एसेट्स को उत्पन्न करने वाले एप्लिकेशन के आधार पर फ़िल्टर या टैग करती हैं, इसलिए बड़े प्रोजेक्ट्स में यह कदम आवश्यक है।

### चरण 3: मापन इकाइयों को परिभाषित करें

Aspose.3D आपको सीन के लिए यूनिट सिस्टम निर्दिष्ट करने की अनुमति देता है। इस उदाहरण में हम एक प्राचीन मिस्र की इकाई “pole” को कस्टम स्केल फ़ैक्टर के साथ उपयोग करते हैं।

```java
scene.getAssetInfo().setUnitName("pole");
scene.getAssetInfo().setUnitScaleFactor(0.6);
```

> **टिप:** `unitScaleFactor` को अपने मॉडल के वास्तविक आकार के अनुसार समायोजित करें; 1.0 का मतलब चुनी हुई इकाई के साथ 1‑to‑1 मैपिंग है।

### चरण 4: सीन को FBX में एक्सपोर्ट करें

अब जब एसेट जानकारी जुड़ गई है, हम सीन को FBX फ़ाइल के रूप में सहेजते हैं। `FileFormat.FBX7500ASCII` विकल्प एक मानव‑पठनीय ASCII FBX बनाता है, जो डिबगिंग के लिए उपयोगी है।

```java
String MyDir = "Your Document Directory";
MyDir = MyDir + "InformationToScene.fbx";
scene.save(MyDir, FileFormat.FBX7500ASCII);
// ExEnd:AddAssetInformationToScene
```

> **ध्यान रखें:** `"Your Document Directory"` को एक पूर्ण पाथ या अपने प्रोजेक्ट की कार्यशील डायरेक्टरी के सापेक्ष पाथ से बदलें।

### चरण 5: सफलता संदेश प्रिंट करें

एक साधारण कंसोल आउटपुट यह पुष्टि करता है कि ऑपरेशन सफल रहा और फ़ाइल कहाँ लिखी गई, यह बताता है।

```java
System.out.println("\nAsset information added successfully to Scene.\nFile saved at " + MyDir);
```

## सामान्य उपयोग केस

- **गेम एसेट पाइपलाइन** – संस्करण ट्रैकिंग के लिए FBX फ़ाइलों में सीधे निर्माता जानकारी एम्बेड करें।  
- **आर्किटेक्चरल विज़ुअलाइज़ेशन** – प्रोजेक्ट‑विशिष्ट यूनिट्स स्टोर करें ताकि रेंडरिंग इंजन में इम्पोर्ट करते समय स्केलिंग त्रुटियों से बचा जा सके।  
- **ऑटोमेटेड रिपोर्टिंग** – मेटाडेटा के साथ ऑन‑द‑फ़्लाई FBX फ़ाइलें जनरेट करें जिन्हें डाउनस्ट्रीम एनालिटिक्स टूल पढ़ सकते हैं।

## समस्या निवारण एवं टिप्स

| समस्या | समाधान |
|-------|----------|
| **सेव के बाद फ़ाइल नहीं मिली** | सुनिश्चित करें कि `MyDir` मौजूद फ़ोल्डर की ओर इशारा कर रहा है और आपके एप्लिकेशन के पास लिखने की अनुमति है। |
| **बाहरी व्यूअर में यूनिट्स गलत दिख रही हैं** | `unitScaleFactor` को दोबारा जांचें; कुछ व्यूअर बेस यूनिट के रूप में मीटर की अपेक्षा करते हैं। |
| **एसेट मेटाडेटा गायब है** | `scene.save()` से पहले `scene.getAssetInfo()` को कॉल करना सुनिश्चित करें; `save()` के बाद किए गए परिवर्तन स्थायी नहीं होते। |

## अक्सर पूछे जाने वाले प्रश्न (FAQ)

**प्रश्न: आउटपुट फ़ॉर्मेट को बाइनरी FBX में कैसे बदलूँ?**  
उत्तर: `scene.save(...)` कॉल करते समय `FileFormat.FBX7500ASCII` को `FileFormat.FBX7500` से बदल दें।

**प्रश्न: क्या मैं बिल्ट‑इन एसेट फ़ील्ड्स के अलावा कस्टम यूज़र‑डिफाइंड मेटाडेटा जोड़ सकता हूँ?**  
उत्तर: हाँ, `scene.getUserData().add("Key", "Value")` का उपयोग करके अतिरिक्त की‑वैल्यू पेयर एम्बेड करें।

**प्रश्न: क्या Aspose.3D अन्य एक्सपोर्ट फ़ॉर्मेट जैसे OBJ या GLTF को सपोर्ट करता है?**  
उत्तर: करता है। आवश्यकतानुसार `FileFormat` एन्‍युम को `OBJ` या `GLTF2` में बदलें।

**प्रश्न: किस संस्करण का Java आवश्यक है?**  
उत्तर: Aspose.3D for Java Java 8 और उसके बाद के संस्करणों को सपोर्ट करता है।

**प्रश्न: क्या मौजूदा FBX को लोड करके उसकी एसेट जानकारी बदलना और फिर पुनः सहेजना संभव है?**  
उत्तर: बिल्कुल। `new Scene("input.fbx")` से फ़ाइल लोड करें, `scene.getAssetInfo()` को संशोधित करें, फिर सहेजें।
