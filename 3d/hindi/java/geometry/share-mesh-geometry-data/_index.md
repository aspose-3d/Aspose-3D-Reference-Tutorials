# जावा 3D में मेष को FBX में बदलें और सामग्री रंग सेट करें

## परिचय

यदि आप जावा‑आधारित 3D एप्लिकेशन बना रहे हैं, तो अक्सर आपको एक ही ज्योमेट्री को कई ऑब्जेक्ट्स में पुन: उपयोग करना पड़ता है, जबकि प्रत्येक इंस्टेंस को एक अनोखा लुक देना होता है। इस ट्यूटोरियल में आप सीखेंगे **मेश को FBX में कैसे बदलें**, कई नोड्स के बीच मूल मेश ज्योमेट्री को साझा करें, और **प्रत्येक नोड के लिए अलग सामग्री रंग सेट करें**। अंत तक आपके पास एक तैयार‑से‑एक्सपोर्ट FBX सीन होगा जिसे आप किसी भी इंजन या व्यूअर में डाल सकते हैं।

## त्वरित उत्तर
- **मुख्य लक्ष्य क्या है?** मेष को FBX में बदलना, मेष ज्योमेट्री को साझा करना, और प्रत्येक नोड के लिए अलग सामग्री रंग सेट करना।  
- **कौनसी लाइब्रेरी आवश्यक है?** जावा के लिए Aspose.3D।  
- **परिणाम कैसे निर्यात करें?** `FileFormat.FBX7400ASCII` का उपयोग करके सीन को FBX फ़ाइल के रूप में सहेजें।  
- **क्या मुझे लाइसेंस चाहिए?** प्रोडक्शन उपयोग के लिए एक अस्थायी या पूर्ण लाइसेंस आवश्यक है।  
- **कौनसा जावा संस्करण काम करता है?** कोई भी Java 8+ पर्यावरण।

## **convert mesh to FBX** क्या है?

`convert mesh to fbx` वह प्रक्रिया है जिसमें मेमोरी में निर्मित या संशोधित मेष ऑब्जेक्ट को FBX फ़ाइल फ़ॉर्मेट में लिखा जाता है, जिसे Maya, Blender, और Unity जैसे 3D टूल्स व्यापक रूप से समर्थन करते हैं। Aspose.3D भारी काम संभालता है, इसलिए आपको केवल उपयुक्त `FileFormat` के साथ `scene.save(...)` कॉल करने की आवश्यकता है।

## मेष ज्योमेट्री डेटा को साझा क्यों करें?

ज्यामिति को साझा करने से मेमोरी खपत कम होती है और रेंडरिंग तेज़ होती है क्योंकि मूल वर्टेक्स बफ़र केवल एक बार संग्रहीत होते हैं। यह तकनीक उन दृश्यों के लिए परिपूर्ण है जिनमें कई डुप्लिकेट ऑब्जेक्ट्स होते हैं—जैसे जंगल, भीड़, या मॉड्यूलर आर्किटेक्चर—जहाँ प्रत्येक इंस्टेंस केवल ट्रांसफ़ॉर्म या सामग्री में अंतर रखता है।

## पूर्वापेक्षाएँ

ट्यूटोरियल में प्रवेश करने से पहले सुनिश्चित करें कि आपके पास निम्नलिखित पूर्वापेक्षाएँ मौजूद हैं:

- **जावा विकास पर्यावरण** – कोई भी IDE या कमांड‑लाइन सेटअप जिसमें Java 8 या नया हो।  
- **Aspose.3D लाइब्रेरी** – आधिकारिक साइट से नवीनतम JAR डाउनलोड करें: [यहाँ](https://releases.aspose.com/3d/java/)।

## पैकेज आयात करें

अपने जावा प्रोजेक्ट में आवश्यक पैकेज आयात करके शुरू करें। यह कदम Aspose.3D लाइब्रेरी द्वारा प्रदान की गई कार्यक्षमताओं तक पहुँचने के लिए महत्वपूर्ण है।

```java
import com.aspose.threed.*;
```

## चरण 1: सीन ऑब्जेक्ट प्रारंभ करें (initialize scene java)

आइए प्रक्रिया को एक सीन ऑब्जेक्ट को प्रारंभ करके शुरू करें। यह वह कैनवास होगा जहाँ हमारी 3D जादू का प्रदर्शन होगा।

```java
// Initialize scene object
Scene scene = new Scene();
```

## चरण 2: रंग वेक्टर परिभाषित करें

इस चरण में, हम एक रंग वेक्टर की एरे परिभाषित करेंगे जिसे हमारे 3D सीन के विभिन्न तत्वों पर लागू किया जाएगा।

```java
// Define color vectors
Vector3[] colors = new Vector3[] {
    new Vector3(1, 0, 0),
    new Vector3(0, 1, 0),
    new Vector3(0, 0, 1)
};
```

## चरण 3: पॉलीगॉन बिल्डर का उपयोग करके 3D मेष जावा बनाएं (create 3d mesh java)

Common क्लास का उपयोग करके पॉलीगॉन बिल्डर मेथड से मेष बनाएं। यह मेष हमारे 3D तत्वों की नींव होगी।

```java
// Call Common class create mesh using polygon builder method to set mesh instance
Mesh mesh = Common.createMeshUsingPolygonBuilder();
```

## प्रत्येक नोड के लिए सामग्री रंग कैसे सेट करें?

रंग वेक्टर के माध्यम से इटररेट करें, क्यूब नोड बनाएं, और सामग्री, **set material color**, तथा ट्रांसलेशन जैसे एट्रिब्यूट सेट करें। यह प्रत्येक मेष इंस्टेंस की दृश्य उपस्थिति को नियंत्रित करने का मुख्य भाग है।

```java
int idx = 0;
for(Vector3 color : colors) {
    // Initialize cube node object
    Node cube = new Node("cube");
    cube.setEntity(mesh);
    LambertMaterial mat = new LambertMaterial();
    // Set color
    mat.setDiffuseColor(color);
    // Set material
    cube.setMaterial(mat);
    // Set translation
    cube.getTransform().setTranslation(new Vector3(idx++ * 20, 0, 0));
    // Add cube node
    scene.getRootNode().addChildNode(cube);
}
```

## चरण 5: 3D सीन सहेजें (save scene fbx, convert mesh to fbx)

समर्थित फ़ाइल फ़ॉर्मेट, इस मामले में FBX7400ASCII, में 3D सीन को सहेजने के लिए डायरेक्टरी और फ़ाइलनाम निर्दिष्ट करें। यह चरण **convert mesh to FBX** को भी दर्शाता है।

```java
// The path to the documents directory.
String MyDir = "Your Document Directory";
MyDir = MyDir + "MeshGeometryData.fbx";

// Save 3D scene in the supported file formats
scene.save(MyDir, FileFormat.FBX7400ASCII);
```

## सामान्य समस्याएँ और सुझाव

- **पाथ समस्याएँ** – फ़ाइलनाम जोड़ने से पहले सुनिश्चित करें कि डायरेक्टरी पाथ फ़ाइल सेपरेटर (`/` या `\\`) पर समाप्त हो।  
- **लाइसेंस प्रारंभिकरण** – यदि आप `scene.save` कॉल करने से पहले Aspose.3D लाइसेंस सेट करना भूल जाते हैं, तो लाइब्रेरी ट्रायल मोड में काम करेगी और संभवतः वॉटरमार्क एम्बेड कर देगी।  
- **सामग्री ओवरराइट** – कई नोड्स के लिए एक ही `LambertMaterial` इंस्टेंस को पुनः उपयोग करने से सभी नोड्स एक ही रंग साझा करेंगे। हमेशा प्रत्येक इटरेशन में नया सामग्री बनाएं, जैसा कि ऊपर दिखाया गया है।  
- **बड़े मेष** – यदि मेष में लाखों वर्टेक्स हैं, तो FBX फ़ाइल आकार को नियंत्रित रखने के लिए इंडेक्स्ड पॉलीगॉन के साथ `MeshBuilder` उपयोग करने पर विचार करें।

## अक्सर पूछे जाने वाले प्रश्न

**Q: क्या मैं सीन को FBX के अलावा अन्य फ़ॉर्मेट में निर्यात कर सकता हूँ?**  
A: हाँ, Aspose.3D OBJ, STL, 3MF, GLTF आदि को समर्थन देता है। बस `save` कॉल में `FileFormat` एनोम को बदल दें।

**Q: यदि सीन बन जाने के बाद मुझे सामग्री बदलनी पड़े तो क्या करें?**  
A: नोड को प्राप्त करें, उसके `LambertMaterial` को संशोधित करें (जैसे `setDiffuseColor`), और सीन को पुनः सहेजें।

**Q: क्या लाइब्रेरी बड़े मेषों को प्रभावी ढंग से संभालती है?**  
A: Aspose.3D अनुकूलित डेटा स्ट्रक्चर का उपयोग करता है; फिर भी, अत्यधिक बड़े मेषों के लिए स्ट्रीमिंग या सीन को विभाजित करने पर विचार करें।

**Q: क्या रंग परिवर्तन को एनिमेट करने का कोई तरीका है?**  
A: हाँ, आप `AnimationController` API का उपयोग करके सामग्री गुणों को एनिमेट कर सकते हैं।
