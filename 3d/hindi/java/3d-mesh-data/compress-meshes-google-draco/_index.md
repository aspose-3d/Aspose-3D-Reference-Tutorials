# 3D मॉडल आकार घटाएँ: जावा में ड्राको के साथ स्फीयर मेष बनाएं

## परिचय

यदि आप **3D मॉडल आकार घटाने** का तेज़ तरीका खोज रहे हैं जबकि उच्च‑गुणवत्ता वाली ज्योमेट्री प्रदान करना चाहते हैं, तो आप सही जगह पर आए हैं। इस ट्यूटोरियल में हम **Aspose.3D for Java** के साथ एक स्फीयर मेष जनरेट करेंगे और फिर **Google Draco** का उपयोग करके उस मेष को संकुचित करेंगे। अंत में आपके पास एक तैयार‑उपयोग `.drc` फ़ाइल होगी जो मूल से बहुत छोटी होगी, जिससे यह वेब‑आधारित व्यूअर्स, मोबाइल गेम्स, या किसी भी बैंडविड्थ‑सीमित जावा एप्लिकेशन के लिए उपयुक्त बनती है।

## त्वरित उत्तर

- **इस ट्यूटोरियल में क्या कवर किया गया है?** जावा में स्फीयर मेष बनाना और Aspose.3D के माध्यम से Google Draco के साथ उसे संकुचित करना।  
- **मुख्य लाइब्रेरी?** Aspose.3D for Java (मेश निर्माण और Draco निर्यात दोनों के लिए उपयोग की जाती है)।  
- **आम कार्यान्वयन समय?** बेसिक स्फीयर के लिए लगभग 10‑15 मिनट।  
- **मुख्य पूर्वापेक्षा?** क्लासपाथ में Aspose.3D JARs के साथ जावा विकास पर्यावरण।  
- **परिणाम?** एक `.drc` फ़ाइल जो **3D मॉडल आकार घटाती** है, अनकम्प्रेस्ड मेष की तुलना में 90 % तक।

## 3D विकास के संदर्भ में “3D मॉडल आकार घटाना” क्या है?

3D मॉडल आकार घटाने का मतलब है ज्योमेट्री डेटा की मात्रा को कम करना जिसे ट्रांसफ़र या स्टोर करना आवश्यक है, बिना दृश्य गुणवत्ता में स्पष्ट गिरावट के। Draco यह वर्टेक्स पोज़िशन, नॉर्मल्स और अन्य एट्रिब्यूट्स को अत्यधिक कॉम्पैक्ट बाइनरी फ़ॉर्मेट में एन्कोड करके करता है। जब इसे Aspose.3D के साथ जोड़ा जाता है, तो पूरा वर्कफ़्लो जावा के भीतर रहता है, इसलिए आपको नेटिव बाइनरीज़ को संभालने की ज़रूरत नहीं पड़ती।

## Aspose.3D के साथ Google Draco मेष संपीड़न क्यों उपयोग करें?

- **विनाशकारी आकार घटाव:** Draco मेष डेटा को OBJ या STL जैसे फ़ॉर्मेट की तुलना में 90 % तक घटा सकता है।  
- **तेज़ रनटाइम डिकोडिंग:** Unity, Unreal, और three.js जैसे इंजन Draco को नेटिव रूप से डिकोड करते हैं, जिससे लोड समय तेज़ हो जाता है।  
- **सहज जावा इंटीग्रेशन:** Aspose.3D नेटिव Draco लाइब्रेरी को एब्स्ट्रैक्ट करता है, जिससे आप जावा इकोसिस्टम में ही रह सकते हैं।  
- **एक‑स्टॉप Aspose 3D निर्यात:** वही API जिसका उपयोग आप ज्योमेट्री बनाने के लिए करते हैं, निर्यात को भी संभालता है, जिससे पाइपलाइन सरल हो जाती है।

## पूर्वापेक्षाएँ

- **Java Development Kit (JDK)** – संस्करण 8 या नया।  
- **Aspose.3D for Java** – आधिकारिक पेज से नवीनतम JARs डाउनलोड करें [here](https://releases.aspose.com/3d/java/).  
- **Google Draco की बुनियादी परिचितता** – आप Aspose.3D के रैपर का उपयोग करेंगे, इसलिए नेटिव Draco सेटअप की आवश्यकता नहीं है।

## पैकेज आयात करें

```java
import com.aspose.threed.DracoCompressionLevel;
import com.aspose.threed.DracoSaveOptions;
import com.aspose.threed.FileFormat;
import com.aspose.threed.Sphere;


import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Paths;
```

## स्टेप‑बाय‑स्टेप गाइड

### स्टेप 1: प्रोजेक्ट सेट अप करें

एक नया जावा प्रोजेक्ट बनाएं (कोई भी IDE काम करेगा) और सभी Aspose.3D JARs को क्लासपाथ में जोड़ें। स्पष्टता के लिए अपने स्रोत फ़ाइलों को `com.example.draco` जैसे पैकेज में रखें।

### स्टेप 2: जावा में स्फीयर मेष कैसे बनाएं

```java
// ExStart:Encode3DMeshinGoogleDraco
// The path to the documents directory.
String MyDir = "Your Document Directory";

// Create a sphere
Sphere sphere = new Sphere();
```

> **प्रो टिप:** `Sphere` क्लास डिफ़ॉल्ट रेडियस 1.0 के साथ एक त्रिकोणीय मेष जनरेट करती है। यदि आपको संपीड़न से पहले अलग स्तर का विवरण चाहिए तो आप कस्टम रेडियस, टेस्सेलेशन, या मैटेरियल पैरामीटर पास कर सकते हैं।

### स्टेप 3: Google Draco के साथ मेष कैसे संपीड़ित करें

```java
// Encode the sphere to Google Draco raw data using optimal compression level.
DracoSaveOptions opt = new DracoSaveOptions();
opt.setCompressionLevel(DracoCompressionLevel.OPTIMAL);
byte[] b = FileFormat.DRACO.encode(sphere.toMesh(), opt);
```

कम्प्रेशन लेवल को `OPTIMAL` सेट करने से सबसे बड़ा आकार घटाव मिलता है जबकि दृश्य सटीकता बनी रहती है, जिससे आप सीधे **3D मॉडल आकार घटा** सकते हैं।

### स्टेप 4: संपीड़ित मेष को सहेजें

```java
// Save the raw bytes to file
Files.write(Paths.get(MyDir, "SphereMeshtoDRC_Out.drc"), b);
// ExEnd:Encode3DMeshinGoogleDraco
```

परिणामी `SphereMeshtoDRC_Out.drc` को क्लाइंट्स को स्ट्रीम किया जा सकता है, CDN में संग्रहीत किया जा सकता है, या किसी भी Draco‑संगत इंजन द्वारा सीधे लोड किया जा सकता है।

## सामान्य समस्याएँ और समाधान

| समस्या | कारण | समाधान |
|-------|--------|-----|
| **`NoClassDefFoundError` for Draco classes** | Aspose.3D JARs क्लासपाथ में नहीं हैं | सुनिश्चित करें कि *सभी* Aspose.3D JAR फ़ाइलें शामिल हैं और संस्करण दस्तावेज़ीकरण से मेल खाता है। |
| **Output file is empty** | `MyDir` एक गैर‑मौजूद फ़ोल्डर की ओर इशारा करता है | फ़ाइल लिखने से पहले प्रोग्रामेटिकली डायरेक्टरी बनाएं (`Files.createDirectories(Paths.get(MyDir))`)। |
| **Compressed mesh looks distorted** | कम कम्प्रेशन लेवल या अपर्याप्त टेस्सेलेशन का उपयोग | `DracoCompressionLevel.OPTIMAL` पर स्विच करें और स्फीयर की टेस्सेलेशन बढ़ाएँ (उदा., `new Sphere(1.0, 64, 64)`)। |
