# Aspose.3D for .NET का उपयोग करके कस्टम बाइनरी फॉर्मेट में 3D मेष सहेजें

## परिचय

स्वागत है! इस **Aspose 3D ट्यूटोरियल** में आप सीखेंगे कि कैसे **3D मेष** डेटा को एक कस्टम बाइनरी फॉर्मेट में **सहेजा** जाए। चाहे आपको गेम इंजन के लिए **FBX बाइनरी** फ़ाइलों को बदलने की आवश्यकता हो या अपना हल्का मेष कंटेनर बनाना हो, यह गाइड स्पष्ट C# उदाहरणों के साथ आपको हर चरण में ले जाता है। निर्देश मानते हैं कि आप बुनियादी C# सिंटैक्स में सहज हैं और आपके पास Aspose.3D की कार्यशील इंस्टॉलेशन है।

## त्वरित उत्तर

- **इस ट्यूटोरियल में क्या कवर किया गया है?** Aspose.3D for .NET के साथ एक कस्टम बाइनरी फ़ाइल में 3D मेष को सहेजना।  
- **कौन से फ़ाइल फॉर्मेट बदल सकते हैं?** कोई भी फ़ॉर्मेट जो Aspose.3D पढ़ता है (जैसे, FBX, OBJ, 3DS) – हम एक FBX स्रोत के साथ प्रदर्शन करेंगे।  
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए एक मुफ्त ट्रायल काम करता है; उत्पादन के लिए एक व्यावसायिक लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+।  
- **कार्यान्वयन में कितना समय लगेगा?** बुनियादी रूपांतरण के लिए लगभग 10‑15 मिनट।

## **save 3d mesh** क्या है?

3D मेष को सहेजना का मतलब है कि सीन से वर्टेक्स पोजीशन, नॉर्मल्स, UV कोऑर्डिनेट्स, और ट्रायंगल इंडेक्स को निकालना और उन्हें एक फ़ाइल में लिखना जिसे आप परिभाषित करते हैं। एक कस्टम बाइनरी फॉर्मेट आपको स्टोरेज आकार और पढ़ने के प्रदर्शन पर पूर्ण नियंत्रण देता है, जो रियल‑टाइम रेंडरिंग या नेटवर्क ट्रांसमिशन के लिए आवश्यक है।

## क्यों **convert FBX binary** और **create custom mesh format**?

- **प्रदर्शन:** बाइनरी डेटा टेक्स्ट‑आधारित फॉर्मेट जैसे OBJ की तुलना में तेज़ लोड होता है।  
- **पोर्टेबिलिटी:** एक कस्टम फॉर्मेट को आपके इंजन की सटीक आवश्यकताओं के अनुसार तैयार किया जा सकता है, जिससे अनावश्यक डेटा हटाया जा सकता है।  
- **सुरक्षा:** बाइनरी फ़ाइलें आकस्मिक संपादन के कम प्रवण होती हैं, जिससे स्वामित्व वाली ज्योमेट्री की सुरक्षा में मदद मिलती है।  

Aspose.3D का उपयोग करके रूपांतरण सीधा हो जाता है जबकि कोड साफ़ और रखरखाव योग्य रहता है।

- इनपुट 3D फ़ाइल: एक 3D फ़ाइल (जैसे, test.fbx) रखें जिसे आप कस्टम बाइनरी फॉर्मेट में बदलना चाहते हैं।

## नेमस्पेस आयात करें

अपने C# कोड में, Aspose.3D कार्यक्षमताओं तक पहुँचने के लिए आवश्यक नेमस्पेस शामिल करें:

```csharp
using Aspose.ThreeD;
using Aspose.ThreeD.Entities;
using Aspose.ThreeD.Utilities;
using System;
using System.Collections.Generic;
using System.IO;
using System.Linq;
using System.Text;
```

ये नेमस्पेस आपको सीन हैंडलिंग, मेष रूपांतरण उपयोगिताएँ, और बुनियादी .NET I/O क्लासेज़ तक पहुँच प्रदान करते हैं।

## चरण 1: 3D फ़ाइल लोड करें

Aspose.3D का उपयोग करके अपनी 3D फ़ाइल लोड करें। इस उदाहरण में, हम **test.fbx** नामक फ़ाइल लोड करते हैं:

```csharp
Scene scene = Scene.FromFile("test.fbx");
```

`Scene.FromFile` मेथड स्वचालित रूप से स्रोत फॉर्मेट का पता लगाता है, इसलिए आप FBX फ़ाइल को OBJ, 3DS, या किसी अन्य समर्थित प्रकार से बदल सकते हैं।

## चरण 2: कस्टम बाइनरी फॉर्मेट परिभाषित करें

उस कस्टम बाइनरी फॉर्मेट की संरचना परिभाषित करें जिसमें आप अपने 3D मेष सहेजना चाहते हैं। उदाहरण में `MeshBlock`, `Vertex`, और `Triangle` को घटकों के रूप में उपयोग किया गया है।

```csharp
//The memory layout of a vertex is 
// float[3] position;
// float[3] normal;
// float[3] uv;
var vertexDeclaration = new VertexDeclaration();
vertexDeclaration.AddField(VertexFieldDataType.FVector3, VertexFieldSemantic.Position);
vertexDeclaration.AddField(VertexFieldDataType.FVector3, VertexFieldSemantic.Normal);
vertexDeclaration.AddField(VertexFieldDataType.FVector3, VertexFieldSemantic.UV);
```

वर्टेक्स लेआउट घोषित करके आप Aspose.3D को बताते हैं कि बाइनरी स्ट्रीम में लिखने से पहले डेटा को कैसे पैक किया जाए।

## चरण 3: लिखने के लिए फ़ाइल खोलें

एक बाइनरी फ़ाइल खोलें लिखने के लिए, जहाँ परिवर्तित 3D मेष सहेजे जाएंगे:

```csharp
using (var writer = new BinaryWriter(new FileStream("Your Output Directory" + "Save3DMeshesInCustomBinaryFormat_out", FileMode.Create, FileAccess.Write)))
```

`BinaryWriter` आपको बाइट क्रम पर लो‑लेवल नियंत्रण देता है और सुनिश्चित करता है कि फ़ाइल प्रत्येक रन पर नई बनाई जाए।

## चरण 4: नोड्स और एंटिटीज़ पर इटररेट करें

3D सीन में प्रत्येक नोड पर जाएँ और मेष एंटिटीज़ को कस्टम बाइनरी फॉर्मेट में बदलें। लाइट्स, कैमरों और अन्य गैर‑मेश एंटिटीज़ को अनदेखा करें:

```csharp
scene.RootNode.Accept(delegate(Node node)
{
    foreach (Entity entity in node.Entities)
    {
        if (!(entity is IMeshConvertible))
            continue;
        // ... (continue processing)
    }
    return true;
});
```

`Accept` मेथड एक डेप्थ‑फ़र्स्ट ट्रैवर्सल करता है, जिससे आप हायरार्की की गहराई की परवाह किए बिना प्रत्येक मेष को संभाल सकते हैं।

## चरण 5: कंट्रोल पॉइंट्स और ट्रायंगल्स को बदलें और लिखें

प्रत्येक मेष एंटिटी के लिए, कंट्रोल पॉइंट्स को वर्ल्ड स्पेस में बदलें और उन्हें ट्रायंगल इंडेक्स के साथ बाइनरी फ़ाइल में लिखें:

```csharp
Mesh m = ((IMeshConvertible)entity).ToMesh();

var triMesh = TriMesh.FromMesh(vertexDeclaration, m);


//The mesh's memory layout is:
// float[16] transform_matrix;
// int vertices_count;
// int indices_count;
// vertex[vertices_count] vertices;
// ushort[indices_count] indices;


//write transform
var transform = node.GlobalTransform.TransformMatrix.ToArray();
for(int i = 0; i < transform.Length; i++)
    writer.Write((float)transform[i]);
//write number of vertices/indices
writer.Write(triMesh.VerticesCount);
writer.Write(triMesh.IndicesCount);
//write vertices and indices
writer.Flush();
triMesh.WriteVerticesTo(writer.BaseStream);
triMesh.Write16bIndicesTo(writer.BaseStream);
```

यह ब्लॉक पहले नोड के वर्ल्ड‑स्पेस ट्रांसफ़ॉर्म मैट्रिक्स को लिखता है, उसके बाद वर्टेक्स काउंट, इंडेक्स काउंट, वर्टेक्स बफ़र, और अंत में 16‑बिट इंडेक्स बफ़र। परिणामी फ़ाइल को कोई भी इंजन जो इस लेआउट को जानता है, कुशलता से पढ़ सकता है।

## सामान्य समस्याएँ और समाधान

- **फ़ाइल पाथ त्रुटियाँ:** सुनिश्चित करें कि आउटपुट डायरेक्टरी मौजूद है या वैध पाथ बनाने के लिए `Path.Combine` का उपयोग करें।  
- **बड़े मेष:** लाखों वर्टेक्स वाले मेष के लिए, `OutOfMemoryException` से बचने के लिए चंक्स में लिखने पर विचार करें।  
- **कोऑर्डिनेट सिस्टम मिसमैच:** Aspose.3D एक राइट‑हैंडेड कोऑर्डिनेट सिस्टम उपयोग करता है; यदि आपके लक्ष्य इंजन को लेफ़्ट‑हैंडेड चाहिए तो बदलें।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या यह तरीका एनिमेटेड मेष के साथ काम करता है?**  
**उत्तर:** हाँ, आप बाइनरी डेटा लिखने से पहले एनीमेशन की कीफ़्रेम्स पर इटररेट करके प्रत्येक फ्रेम के ट्रांसफ़ॉर्म्ड वर्टेक्स को एक्सपोर्ट कर सकते हैं।

**प्रश्न: क्या मैं बोन वेट्स जैसे कस्टम वर्टेक्स एट्रिब्यूट्स जोड़ सकता हूँ?**  
**उत्तर:** बिल्कुल। `VertexDeclaration` को अतिरिक्त फ़ील्ड्स (जैसे, `VertexFieldSemantic.BoneWeight`) के साथ विस्तारित करें और मानक वर्टेक्स ब्लॉक के बाद अतिरिक्त डेटा लिखें।

**प्रश्न: कस्टम बाइनरी फ़ाइल को सीन में वापस पढ़ने का सबसे अच्छा तरीका क्या है?**  
**उत्तर:** एक मेल खाने वाला बाइनरी रीडर लागू करें जो ट्रांसफ़ॉर्म मैट्रिक्स, वर्टेक्स काउंट, इंडेक्स काउंट पढ़े, फिर `TriMesh.FromBinary` का उपयोग करके `TriMesh` को पुनः बनाता है।
