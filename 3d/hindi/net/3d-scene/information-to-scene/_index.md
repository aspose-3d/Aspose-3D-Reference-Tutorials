# मेटाडेटा कैसे जोड़ें: दृश्य एसेट्स में जानकारी निकालना

## परिचय

इस व्यापक ट्यूटोरियल में आप Aspose.3D for .NET के साथ अपनी 3D सीन में **मेटाडेटा कैसे जोड़ें** की खोज करेंगे। विक्रेता विवरण, कस्टम माप इकाइयाँ और अन्य एसेट जानकारी जैसे मेटाडेटा जोड़ने से आपके मॉडल अधिक सूचनात्मक, खोज योग्य और गेम इंजन या AR/VR प्लेटफ़ॉर्म जैसे डाउनस्ट्रीम पाइपलाइन के लिए तैयार हो जाते हैं।

## त्वरित उत्तर
- **प्राथमिक उद्देश्य क्या है?** 3D सीन में सीधे मेटाडेटा (विक्रेता जानकारी, इकाइयाँ, कस्टम टैग) एम्बेड करना।  
- **कौन सी लाइब्रेरी उपयोग की जाती है?** Aspose.3D for .NET।  
- **क्या मेटाडेटा जोड़ने के बाद मॉडल निर्यात कर सकता हूँ?** हाँ – आप **3D मॉडल निर्यात** कर सकते हैं, जैसे FBX के रूप में सहेजें।  
- **क्या मुझे लाइसेंस चाहिए?** एक मुफ्त ट्रायल उपलब्ध है; उत्पादन के लिए एक व्यावसायिक लाइसेंस आवश्यक है।  
- **कौन से .NET संस्करण समर्थित हैं?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6।

## 3D सीन में “मेटाडेटा कैसे जोड़ें” क्या है?

मेटाडेटा जोड़ना का अर्थ है वर्णनात्मक जानकारी—जैसे एप्लिकेशन नाम, विक्रेता, माप इकाइयाँ, या कस्टम टैग—को सीधे सीन फ़ाइल में संलग्न करना। यह डेटा **save scene as FBX** या अन्य समर्थित फ़ॉर्मेट में सहेजते समय मॉडल के साथ यात्रा करता है, जिससे डाउनस्ट्रीम टूल्स को बाहरी फ़ाइलों की आवश्यकता के बिना संदर्भ समझ में आता है।

## कस्टम मेटाडेटा और विक्रेता जानकारी एम्बेड क्यों करें?

- **खोज योग्यता:** एसेट मैनेजमेंट सिस्टम विक्रेता या इकाई प्रकार के आधार पर मॉडल फ़िल्टर कर सकते हैं।  
- **परस्पर कार्यक्षमता:** FBX पढ़ने वाले इंजन स्वचालित रूप से सही स्केल लागू कर सकते हैं यदि आप **माप इकाइयाँ परिभाषित** करते हैं।  
- **ब्रांडिंग:** एप्लिकेशन नाम और विक्रेता को शामिल करने से स्वामित्व और लाइसेंस अनुपालन मजबूत होता है।  

## पूर्वापेक्षाएँ

डाइव करने से पहले सुनिश्चित करें कि आपके पास है:

- Aspose.3D for .NET स्थापित। आप इसे [Aspose.3D for .NET page](https://releases.aspose.com/3d/net/) से डाउनलोड कर सकते हैं।

## नेमस्पेसेस इम्पोर्ट करें

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
```

## चरण 1: 3D सीन को इनिशियलाइज़ करें

```csharp
Scene scene = new Scene();
```

एक नया `Scene` ऑब्जेक्ट बनाएं जो सभी ज्योमेट्री और एसेट जानकारी रखेगा।

## चरण 2: एप्लिकेशन सेट करें और **विक्रेता जानकारी जोड़ें**

```csharp
scene.AssetInfo.ApplicationName = "Egypt";
scene.AssetInfo.ApplicationVendor = "Manualdesk";
```

यहां हम **एप्लिकेशन नाम** और **विक्रेता जानकारी** एम्बेड करते हैं। यह **मेटाडेटा कैसे जोड़ें** का मुख्य भाग है जो मॉडल के स्रोत को पहचानता है।

## चरण 3: सटीक स्केलिंग के लिए **माप इकाइयाँ परिभाषित करें**

```csharp
scene.AssetInfo.UnitName = "pole";
scene.AssetInfo.UnitScaleFactor = 0.6;
```

`UnitName` और `UnitScaleFactor` निर्दिष्ट करके आप डाउनस्ट्रीम टूल्स को बताते हैं कि एक “pole” 0.6 मीटर (60 सेमी) के बराबर है। यह चरण आवश्यक है जब आप बाद में **3D मॉडल निर्यात** करते हैं ताकि वास्तविक दुनिया के आयाम सही रहें।

## चरण 4: **सीन को FBX के रूप में सहेजें** – मेटाडेटा के साथ **3D मॉडल निर्यात** करें

```csharp
var output = "Your Output Directory" + "InformationToScene.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

`Save` कॉल सीन को एक FBX फ़ाइल में लिखता है, जिसमें हमने जोड़ा गया सभी मेटाडेटा एम्बेड होता है। यह **मेटाडेटा कैसे जोड़ें** और फिर **सीन को FBX के रूप में सहेजें** को दर्शाता है, जिससे पूरी एसेट जानकारी के साथ **3D मॉडल निर्यात** प्रभावी रूप से होता है।

## चरण 5: सफलता संदेश प्रदर्शित करें

```csharp
Console.WriteLine("\nAsset information added successfully to Scene.\nFile saved at " + output);
```

एक मित्रवत कंसोल संदेश पुष्टि करता है कि मेटाडेटा लिख दिया गया है और फ़ाइल स्थान।

## सामान्य समस्याएँ और सुझाव

- **गलत इकाई स्केल:** `UnitScaleFactor` वास्तविक रूपांतरण से मेल खाता है या नहीं, दोबारा जांचें; अन्यथा निर्यातित मॉडल बहुत बड़े या छोटे दिख सकते हैं।  
- **विक्रेता जानकारी गायब:** यदि `ApplicationVendor` खाली छोड़ दिया जाता है, तो कुछ व्यूअर्स “अज्ञात” दिखाएंगे। संभव हो तो हमेशा इसे सेट करें।  
- **फ़ाइल पाथ त्रुटियाँ:** सुनिश्चित करें कि आउटपुट डायरेक्टरी मौजूद है; अन्यथा `scene.Save` अपवाद फेंकेगा।
