## 일반적인 문제 및 해결책

MaterialConverter는 레거시 소재 정의를 Physically Based Rendering(PBR) 형식으로 변환하는 유틸리티 클래스입니다.

**Issue:** *변환 후 소재가 텍스처를 잃습니다.*  
**Solution:** 저장하기 전에 `MaterialConverter.convertToPbr()`를 사용하세요; 텍스처 맵을 보존하고 셰이더 파라미터를 자동으로 업데이트합니다.

LoadOptions는 Aspose.3D가 입력 파일을 파싱하는 방식을 제어하는 구성 클래스입니다.

**Issue:** *대용량 OBJ 파일에서 OutOfMemoryError가 발생합니다.*  
**Solution:** `LoadOptions.setEnableMemoryOptimization(true)`로 스트리밍을 활성화하고 JVM 힙 크기(`-Xmx4g`)를 늘리세요.

SaveOptions는 씬을 저장할 때 형식, 압축 및 기능 포함과 같은 출력 설정을 구성합니다.

**Issue:** *내보낸 FBX 파일에 애니메이션 데이터가 없습니다.*  
**Solution:** `SaveOptions.setExportAnimations(true)`를 설정하고 원본 씬에 키프레임 애니메이션 트랙이 포함되어 있는지 확인하세요.

## 자주 묻는 질문

**Q:** *Aspose.3D를 상업용 Java 애플리케이션에서 사용할 수 있나요?*  
**A:** 예. 프로덕션 사용을 위해서는 유효한 Aspose 라이선스가 필요하며, 평가용으로 무료 체험을 제공합니다.

**Q:** *로드 및 저장을 지원하는 3D 파일 형식은 무엇인가요?*  
**A:** Aspose.3D는 OBJ, STL, FBX, 3DS, glTF, PLY, U3D, X 등 여러 형식을 지원하며, 총 30개 이상의 형식을 지원합니다.

**Q:** *전체 씬을 다시 작성하지 않고 소재를 PBR로 업그레이드하려면 어떻게 해야 하나요?*  
**A:** API에서 제공하는 `MaterialConverter.convertToPbr()` 메서드를 사용하면 한 번의 호출로 모든 소재를 업데이트합니다.

**Q:** *대형 씬을 저장할 때 파일 크기를 줄이는 방법이 있나요?*  
**A:** 예. `SaveOptions`를 구성하여(예: 압축 활성화, 메쉬 정밀도 감소) 출력 크기를 최적화할 수 있습니다.

**Q:** *Aspose.3D를 사용하기 위해 추가적인 종속성이 필요합니까?*  
**A:** Aspose.3D for Java JAR만 필요하며, 다른 모든 종속성은 포함되어 있습니다.
