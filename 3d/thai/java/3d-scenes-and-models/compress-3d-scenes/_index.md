# ลดขนาดไฟล์ 3D – บีบอัดฉากด้วย Aspose.3D for Java

## บทนำ

หากคุณกำลังส่งมอบทรัพยากร 3D ผ่านเว็บ, อีเมล, หรือจัดเก็บในคลาวด์บัคเก็ต, ขนาดไฟล์ที่ใหญ่สามารถเป็นคอขวดได้อย่างรวดเร็ว  ในบทแนะนำนี้คุณจะได้เรียนรู้ **วิธีลดขนาดไฟล์ 3D** โดยการบีบอัดฉาก 3D ด้วย Aspose.3D for Java เราจะเดินผ่านการสร้างฉาก, การเพิ่มวัตถุ, การปรับการแปลง, และสุดท้ายการบันทึกฉากพร้อมตัวเลือกการบีบอัดที่รักษาคุณภาพภาพไว้ในขณะที่ทำให้ไฟล์เล็กลงอย่างมาก

## คำตอบสั้น
- **“ลดขนาดไฟล์ 3d” หมายถึงอะไร?** หมายถึงการใช้เทคนิคการบีบอัดกับไฟล์ 3‑D เพื่อให้ขนาดบนดิสก์เล็กลงโดยไม่สูญเสียความแม่นยำของเรขาคณิตหรือเทกซ์เจอร์  
- **ฟอร์แมตใดที่รองรับการบีบอัดใน Aspose.3D?** ฟอร์แมต AMF (Additive Manufacturing File) โดยใช้ `AmfSaveOptions`  
- **ต้องมีลิขสิทธิ์เพื่อบีบอัดหรือไม่?** เวอร์ชันทดลองใช้ได้สำหรับการพัฒนา; ต้องมีลิขสิทธิ์เชิงพาณิชย์สำหรับการใช้งานจริง  
- **การบีบอัดเป็นแบบสูญเสียหรือไม่?** ใช่, การบีบอัดในตัวของ Aspose.3D เป็นแบบไม่สูญเสียสำหรับเรขาคณิตและเทกซ์เจอร์  
- **คาดว่าจะลดขนาดได้เท่าไหร่?** ปกติ 30‑60 % ขึ้นอยู่กับความซับซ้อนของฉากและจำนวนเทกซ์เจอร์

## การบีบอัดฉากใน Aspose.3D คืออะไร?
การบีบอัดฉากคือกระบวนการเข้ารหัสฉาก 3‑D ให้เป็นรูปแบบที่กะทัดรัดมากขึ้น Aspose.3D ใช้การบีบอัดแบบ gzip ที่มีอยู่ในฟอร์แมต AMF ทำให้คุณสามารถส่งโมเดลขนาดใหญ่ได้อย่างมีประสิทธิภาพมากขึ้น

## ทำไมต้องลดขนาดไฟล์ 3D?
- **ดาวน์โหลดเร็วขึ้น** – ผู้ใช้ที่มีแบนด์วิธจำกัดจะได้รับประสบการณ์ที่ราบรื่นกว่า  
- **ลดต้นทุนการจัดเก็บ** – ค่าธรรมเนียมคลาวด์คำนวณตาม GB  
- **ประสิทธิภาพดีขึ้น** – ไฟล์เล็กลงโหลดเร็วขึ้นในเบราว์เซอร์และเอนจินเกม  
- **แชร์ง่ายขึ้น** – แนบไฟล์อีเมลและแพลตฟอร์มการทำงานร่วมกันมักมีขีดจำกัดขนาดไฟล์

## ข้อกำหนดเบื้องต้น
ก่อนเริ่ม, ตรวจสอบให้แน่ใจว่าคุณมี:

- Java Development Kit (JDK) 8 หรือใหม่กว่า  
- ไลบรารี Aspose.3D for Java ที่ดาวน์โหลดจากเว็บไซต์ทางการ – คุณสามารถหา link ดาวน์โหลดได้ [ที่นี่](https://releases.aspose.com/3d/java/)  
- IDE สำหรับ Java (IntelliJ IDEA, Eclipse, หรือ VS Code) เพื่อสร้างและรันโปรเจกต์ตัวอย่าง

## นำเข้าแพ็กเกจ
เพิ่มคลาส Aspose.3D ที่จำเป็นลงในไฟล์ซอร์ส Java ของคุณ:

```java
import com.aspose.threed.AmfSaveOptions;
import com.aspose.threed.Box;
import com.aspose.threed.Scene;
import com.aspose.threed.Transform;
import com.aspose.threed.Vector3;
```

## คู่มือแบบขั้นตอน

### ขั้นตอนที่ 1: ตั้งค่าโปรเจกต์ Java ของคุณ
สร้างโปรเจกต์ Java ใหม่ใน IDE ที่คุณชอบและเพิ่มไฟล์ JAR ของ Aspose.3D ลงใน classpath ของโปรเจกต์ เพื่อให้คอมไพเลอร์สามารถหาคลาสที่นำเข้ามาได้

### ขั้นตอนที่ 2: เริ่มต้นฉาก 3D ใหม่
เริ่มด้วยการสร้างอ็อบเจ็กต์ฉากเปล่า คลาส `Scene` เป็นคอนเทนเนอร์สำหรับเรขาคณิต, แสง, กล้อง, และโครงสร้างลำดับชั้นทั้งหมด

```java
// The path to the documents directory.
String MyDir = "Your Document Directory";

Scene scene = new Scene();
```

### ขั้นตอนที่ 3: เพิ่มเรขาคณิตกล่องง่าย ๆ
เพื่อสาธิต, เราจะเพิ่ม primitive กล่องลงในฉาก คลาส `Box` สร้างลูกบาศก์ที่เราสามารถแปลงได้

```java
Box box = new Box();
Transform tr = scene.getRootNode().createChildNode(box).getTransform();
tr.setScale(new Vector3(12, 12, 12));
tr.setTranslation(new Vector3(10, 0, 0));
```

### ขั้นตอนที่ 4: ปรับแต่งกล่อง (สเกล, การหมุน, ตำแหน่ง)
คุณสามารถแก้ไขกล่องเดียวกันหรือเพิ่มอินสแตนซ์เพิ่มเติม ด้านล่างเราจะเปลี่ยนสเกลและใช้มุมออยเลอร์เพื่อหมุนวัตถุ

```java
tr = scene.getRootNode().createChildNode(box).getTransform();
tr.setScale(new Vector3(5, 5, 5));
tr.setEulerAngles(new Vector3(50, 10, 0));
```

### ขั้นตอนที่ 5: บันทึกฉากพร้อมเปิดใช้งานการบีบอัด
กุญแจสำคัญในการ **ลดขนาดไฟล์ 3d** อยู่ที่ `AmfSaveOptions` ตั้งค่า `setEnableCompression(true)` เพื่อเปิดใช้งานการบีบอัด gzip ภายในไฟล์ AMF

```java
AmfSaveOptions opt = new AmfSaveOptions();
opt.setEnableCompression(true);   // Turn on compression to shrink file size
scene.save(MyDir + "compressed_scene.amf", opt);
```

> **เคล็ดลับ:** หากคุณต้องการเก็บเวอร์ชันที่ไม่ได้บีบอัดไว้สำหรับการดีบัก, ให้บันทึกสำเนาที่สองด้วย `setEnableCompression(false)`

ทำซ้ำขั้นตอนข้างต้นสำหรับวัตถุเพิ่มเติมที่คุณต้องการใส่ในฉาก ทุกวัตถุจะถูกเก็บในคอนเทนเนอร์บีบอัดเดียวกัน ทำให้ขนาดไฟล์โดยรวมต่ำลง

## ปัญหาที่พบบ่อย & วิธีแก้
| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|-------|-----|
| **ไฟล์ที่บันทึกยังใหญ่** | การบีบอัดถูกปิดหรือใช้ฟอร์แมตที่ไม่รองรับ (เช่น OBJ) | ตรวจสอบให้แน่ใจว่า `opt.setEnableCompression(true)` และบันทึกเป็น **AMF** |
| **เทกซ์เจอร์ไม่แสดงหลังโหลด** | เทกซ์เจอร์ไม่ได้ฝัง; พาธเป็นไฟล์ภายนอก | ใช้ `scene.getRootNode().getMaterial().setTexture(...).setEmbed(true)` |
| **OutOfMemoryError กับฉากขนาดใหญ่** | โหลดฉากทั้งหมดเข้าสู่หน่วยความจำก่อนบันทึก | เปิดโหมดสตรีมมิงด้วย `AmfSaveOptions.setEnableStreaming(true)` |
