# วิธีบันทึก FBX และดึงข้อมูลฉาก 3D ใน Java

## บทนำ

หคุณกำลังมองหาคู่มือที่ชัดเจนและทำตามได้จริงเกี่ยวกับ **วิธีบันทึกไฟล์ fbx** พร้อมการสกัดข้อมูลเมตาดาต้าที่เป็นประโยชน์จากฉาก 3D ของคุณ คุณมาถูกที่แล้ว ในบทแนะนำนี้เราจะเดินผ่านทุกขั้นตอนโดยใช้ไลบรารี **Aspose.3D Java**: ตั้งแต่การสร้างฉาก, **การตั้งชื่อแอปพลิเคชัน**, **การกำหนดหน่วยวัด**, จนถึงการ **ส่งออกฉากเป็น FBX** สุดท้ายคุณจะได้ไฟล์ FBX ที่พร้อมใช้งานพร้อมข้อมูลสินทรัพย์ที่จำเป็นสำหรับกระบวนการต่อไป

## คำตอบสั้น
- **เป้าหมายหลักคืออะไร?** บันทึกไฟล์ FBX ที่มีข้อมูลสินทรัพย์แบบกำหนดเอง  
- **ใช้ไลบรารีอะไร?** Aspose.3D for Java  
- **ต้องมีลิขสิทธิ์หรือไม่?** สามารถใช้เวอร์ชันทดลองฟรีสำหรับการพัฒนา; ต้องมีลิขสิทธิ์เชิงพาณิชย์สำหรับการใช้งานจริง  
- **สามารถเปลี่ยนหน่วยวัดได้หรือไม่?** ได้ – ใช้ `setUnitName` และ `setUnitScaleFactor`  
- **ไฟล์ผลลัพธ์บันทึกไว้ที่ไหน?** ที่พาธที่คุณระบุใน `scene.save(...)`

## ข้อกำหนดเบื้องต้น

ก่อนเริ่มทำงาน โปรดตรวจสอบว่าคุณมี:

- ความเข้าใจพื้นฐานในไวยากรณ์ Java  
- **Aspose.3D for Java** ที่ดาวน์โหลดและเพิ่มเข้าในโปรเจกต์ของคุณ (คุณสามารถรับได้จากหน้า) [Aspose 3D download page](https://releases.aspose.com/3d/java/)  
- IDE Java ที่คุณชื่นชอบ (IntelliJ IDEA, Eclipse, NetBeans ฯลฯ) ตั้งค่าอย่างถูกต้อง

## นำเข้าแพ็กเกจ

ในไฟล์ซอร์ส Java ของคุณ ให้นำเข้าคลาสของ Aspose.3D ที่ให้การจัดการฉากและการสนับสนุนรูปแบบไฟล์

```java
import com.aspose.threed.FileFormat;
import com.aspose.threed.Scene;
```

> **เคล็ดลับ:** ควรเก็บรายการ import ให้เหลือน้อยที่สุดเพื่อหลีกเลี่ยงการพึ่งพาที่ไม่จำเป็นและเพิ่มความเร็วในการคอมไพล์

## ขั้นตอนการบันทึกไฟล์ FBX คืออะไร?

ต่อไปนี้เป็นขั้นตอนสั้น ๆ ที่แสดง **วิธีเพิ่มข้อมูลสินทรัพย์** ลงในฉากและจากนั้น **ส่งออกฉากเป็น FBX**

### ขั้นตอนที่ 1: เริ่มต้นสร้างฉาก 3D

แรกเริ่มให้สร้างอ็อบเจกต์ `Scene` ว่างเปล่า ซึ่งจะทำหน้าที่เป็นคอนเทนเนอร์สำหรับเรขาคณิต, แสง, กล้อง และเมตาดาต้าสินทรัพย์ทั้งหมด

```java
// ExStart:AddAssetInformationToScene
Scene scene = new Scene();
```

### ขั้นตอนที่ 2: ตั้งค่าข้อมูลแอปพลิเคชันและผู้จำหน่าย

การเพิ่มเมตาดาต้ากำหนดเองช่วยให้เครื่องมือในขั้นตอนต่อไประบุแหล่งที่มาของไฟล์ได้ ที่นี่เราจะ **ตั้งชื่อแอปพลิเคชัน** และผู้จำหน่ายโดยใช้วัตถุ `AssetInfo`

```java
scene.getAssetInfo().setApplicationName("Egypt");
scene.getAssetInfo().setApplicationVendor("Manualdesk");
```

> **ทำไมต้องทำ:** หลาย ๆ pipeline จะกรองหรือแท็กสินทรัพย์ตามแอปพลิเคชันที่สร้างขึ้น ทำให้ขั้นตอนนี้สำคัญสำหรับโครงการขนาดใหญ่

### ขั้นตอนที่ 3: กำหนดหน่วยวัด

Aspose.3D ให้คุณระบุระบบหน่วยที่ฉากของคุณใช้ ในตัวอย่างนี้เราใช้หน่วยอียิปต์โบราณชื่อ “pole” พร้อมสเกลแฟกเตอร์ที่กำหนดเอง

```java
scene.getAssetInfo().setUnitName("pole");
scene.getAssetInfo().setUnitScaleFactor(0.6);
```

> **เคล็ดลับ:** ปรับ `unitScaleFactor` ให้ตรงกับขนาดจริงของโมเดล; ค่า 1.0 หมายถึงการแมป 1‑ต่อ‑1 กับหน่วยที่เลือก

### ขั้นตอนที่ 4: ส่งออกฉากเป็น FBX

เมื่อข้อมูลสินทรัพย์ถูกแนบแล้ว เราจะบันทึกฉากเป็นไฟล์ FBX ตัวเลือก `FileFormat.FBX7500ASCII` จะสร้างไฟล์ ASCII ที่อ่านได้โดยมนุษย์ ซึ่งสะดวกต่อการดีบัก

```java
String MyDir = "Your Document Directory";
MyDir = MyDir + "InformationToScene.fbx";
scene.save(MyDir, FileFormat.FBX7500ASCII);
// ExEnd:AddAssetInformationToScene
```

> **จำไว้:** แทนที่ `"Your Document Directory"` ด้วยพาธแบบ absolute หรือพาธสัมพันธ์กับไดเรกทอรีทำงานของโปรเจกต์ของคุณ

### ขั้นตอนที่ 5: พิมพ์ข้อความยืนยันสำเร็จ

ข้อความคอนโซลง่าย ๆ จะบอกว่าการดำเนินการสำเร็จและแสดงตำแหน่งที่ไฟล์ถูกเขียน

```java
System.out.println("\nAsset information added successfully to Scene.\nFile saved at " + MyDir);
```

## กรณีการใช้งานทั่วไป

- **Pipeline สินทรัพย์เกม** – ฝังข้อมูลผู้สร้างโดยตรงในไฟล์ FBX เพื่อการติดตามเวอร์ชัน  
- **การแสดงผลสถาปัตยกรรม** – เก็บหน่วยเฉพาะโครงการเพื่อหลีกเลี่ยงข้อผิดพลาดการสเกลเมื่อนำเข้าในเครื่องเรนเดอร์  
- **การรายงานอัตโนมัติ** – สร้างไฟล์ FBX แบบไดนามิกพร้อมเมตาดาต้าที่เครื่องมือวิเคราะห์ต่อไปสามารถอ่านได้

## การแก้ไขปัญหาและเคล็ดลับ

| ปัญหา | วิธีแก้ |
|-------|----------|
| **ไฟล์ไม่พบหลังบันทึก** | ตรวจสอบว่า `MyDir` ชี้ไปยังโฟลเดอร์ที่มีอยู่และแอปพลิเคชันของคุณมีสิทธิ์เขียน |
| **หน่วยแสดงผลไม่ถูกต้องในโปรแกรมดูภายนอก** | ตรวจสอบ `unitScaleFactor` อีกครั้ง; โปรแกรมบางตัวคาดหวังเมตรเป็นหน่วยฐาน |
| **เมตาดาต้าสินทรัพย์หาย** | ตรวจสอบว่าคุณเรียก `scene.getAssetInfo()` **ก่อน** การบันทึก; การเปลี่ยนแปลงหลัง `save()` จะไม่ถูกบันทึก |

## คำถามที่พบบ่อยเพิ่มเติม

**ถาม: จะเปลี่ยนรูปแบบผลลัพธ์เป็น binary FBX ได้อย่างไร?**  
ตอบ: แทนที่ `FileFormat.FBX7500ASCII` ด้วย `FileFormat.FBX7500` เมื่อต้องการเรียก `scene.save(...)`

**ถาม: สามารถเพิ่มเมตาดาต้าผู้ใช้ที่กำหนดเองนอกเหนือจากฟิลด์สินทรัพย์ที่มีอยู่ได้หรือไม่?**  
ตอบ: ได้, ใช้ `scene.getUserData().add("Key", "Value")` เพื่อฝังคู่คีย์‑ค่าเพิ่มเติม

**ถาม: Aspose.3D รองรับรูปแบบการส่งออกอื่น ๆ เช่น OBJ หรือ GLTF หรือไม่?**  
ตอบ: รองรับ. เพียงเปลี่ยนค่า enum `FileFormat` เป็น `OBJ` หรือ `GLTF2` ตามต้องการ

**ถาม: ต้องใช้ Java เวอร์ชันใด?**  
ตอบ: Aspose.3D for Java รองรับ Java 8 ขึ้นไป

**ถาม: สามารถโหลด FBX ที่มีอยู่แล้ว, แก้ไขข้อมูลสินทรัพย์, แล้วบันทึกใหม่ได้หรือไม่?**  
ตอบ: ทำได้. โหลดไฟล์ด้วย `new Scene("input.fbx")`, แก้ไข `scene.getAssetInfo()`, แล้วบันทึก
