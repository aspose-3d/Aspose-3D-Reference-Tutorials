## Questions fréquemment posées

**Q : Puis-je combiner la compression Draco avec la génération de données de maillage dans un même pipeline ?**  
R : Oui. Le flux de travail typique consiste à générer d'abord les données manquantes (normales, tangentes, binormales), puis à appliquer la compression Draco au maillage enrichi.

**Q : La réduction de la taille des fichiers 3D affecte-t-elle les performances d'exécution ?**  
R : Réduire la taille du fichier améliore généralement les temps de chargement et réduit l'utilisation de la mémoire. Lorsqu'elle est combinée à une organisation adéquate des maillages (par ex., division par matériau), elle diminue également le nombre d'appels de dessin, augmentant les performances d'exécution.

**Q : Existe-t-il des limites concernant la taille des maillages pouvant être compressés avec Draco ?**  
R : Draco gère des maillages très volumineux, mais les modèles extrêmement haute‑poly peuvent nécessiter d'ajuster les paramètres de compression (par ex., le nombre de bits de quantification) pour équilibrer taille et qualité.

**Q : Dois-je régénérer les tangentes après la décompression d'un maillage Draco ?**  
R : Non. Draco conserve les attributs de sommet, y compris les tangentes, si elles ont été générées avant la compression.

**Q : Une licence commerciale est‑elle requise pour une utilisation en production ?**  
R : Oui. Bien qu'un essai gratuit vous permette d'explorer les fonctionnalités, une licence Aspose.3D valide est nécessaire pour les déploiements en production.
