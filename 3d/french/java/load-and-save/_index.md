## Problèmes courants et solutions

MaterialConverter est une classe utilitaire qui convertit les définitions de matériaux hérités au format Physically Based Rendering (PBR).

**Issue:** *Les matériaux perdent la texture après conversion.*  
**Solution:** Utilisez `MaterialConverter.convertToPbr()` avant l'enregistrement ; il préserve les cartes de textures et met à jour les paramètres du shader automatiquement.

LoadOptions est une classe de configuration qui contrôle la façon dont Aspose.3D analyse les fichiers d'entrée.

**Issue:** *Les gros fichiers OBJ provoquent une OutOfMemoryError.*  
**Solution:** Activez le streaming via `LoadOptions.setEnableMemoryOptimization(true)` et augmentez la taille du tas JVM (`-Xmx4g`).

SaveOptions configure les paramètres de sortie tels que le format, la compression et l'inclusion de fonctionnalités lors de l'enregistrement d'une scène.

**Issue:** *Les fichiers FBX exportés manquent de données d'animation.*  
**Solution:** Définissez `SaveOptions.setExportAnimations(true)` et vérifiez que la scène source contient des pistes d'animation clés.

## Questions fréquentes

**Q:** *Puis-je utiliser Aspose.3D dans une application Java commerciale ?*  
**A:** Oui. Une licence Aspose valide est requise pour une utilisation en production ; un essai gratuit est disponible pour l'évaluation.

**Q:** *Quels formats de fichiers 3D sont pris en charge pour le chargement et l'enregistrement ?*  
**A:** Aspose.3D prend en charge OBJ, STL, FBX, 3DS, glTF, PLY, U3D, X, et plusieurs autres — plus de 30 formats au total.

**Q:** *Comment mettre à jour les matériaux vers le PBR sans réécrire toute la scène ?*  
**A:** Utilisez la méthode `MaterialConverter.convertToPbr()` fournie dans l'API – elle met à jour tous les matériaux en un seul appel.

**Q:** *Existe-t-il un moyen de réduire la taille du fichier lors de l'enregistrement de scènes volumineuses ?*  
**A:** Oui. Configurez `SaveOptions` (par ex., activez la compression, réduisez la précision du maillage) pour optimiser la taille de sortie.

**Q:** *Ai-je besoin de dépendances supplémentaires pour travailler avec Aspose.3D ?*  
**A:** Seul le JAR Aspose.3D for Java est requis ; toutes les autres dépendances sont incluses.
