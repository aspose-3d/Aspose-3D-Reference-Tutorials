## Domande frequenti

**Q: Posso combinare la compressione Draco con la generazione di dati della mesh in un unico pipeline?**  
**A:** Sì. Il flusso di lavoro tipico è generare prima i dati mancanti (normali, tangenti, binormali), poi applicare la compressione Draco alla mesh arricchita.

**Q: La riduzione delle dimensioni del file 3D influisce sulle prestazioni in runtime?**  
**A:** Ridurre le dimensioni del file solitamente migliora i tempi di caricamento e riduce l'uso della memoria. Quando combinato con una corretta organizzazione delle mesh (ad esempio, divisione per materiale), abbassa anche il numero di draw call, aumentando le prestazioni in runtime.

**Q: Ci sono limitazioni sulle dimensioni delle mesh che possono essere compresse con Draco?**  
**A:** Draco gestisce mesh molto grandi, ma modelli estremamente ad alta densità di poligoni potrebbero richiedere la regolazione delle impostazioni di compressione (ad esempio, i bit di quantizzazione) per bilanciare dimensione e qualità.

**Q: È necessario rigenerare le tangenti dopo la decompressione di una mesh Draco?**  
**A:** No. Draco preserva gli attributi dei vertici, incluse le tangenti, se sono state generate prima della compressione.

**Q: È necessaria una licenza commerciale per l'uso in produzione?**  
**A:** Sì. Sebbene una prova gratuita ti consenta di esplorare le funzionalità, è necessaria una licenza valida di Aspose.3D per le distribuzioni in produzione.
