# Как экспортировать FBX и создавать иерархии узлов в Java с Aspose.3D

## Введение

Если вы ищете понятное пошаговое руководство **по экспорту FBX** из Java‑приложения, вы попали по адресу. В этом уроке мы покажем, как построить иерархии узлов, **добавить меш к узлу**, а затем сохранить сцену в файл FBX с помощью Aspose.3D Java API. Независимо от того, создаёте ли вы простой прототип или готовый к продакшену 3D‑движок, эти приёмы дадут вам полный контроль над графом сцены.

## Быстрые ответы
- **Какова основная цель этого урока?** Показать, как экспортировать FBX после построения иерархий узлов.  
- **Какая библиотека используется?** Aspose.3D for Java.  
- **Нужна ли лицензия?** Для разработки достаточно бесплатной пробной версии; для продакшена требуется коммерческая лицензия.  
- **Какой формат файла получается?** FBX (ASCII 7500).  
- **Можно ли настраивать преобразования узлов?** Да – поддерживаются перемещение, вращение и масштабирование.

## Что означает «как экспортировать FBX» в контексте Aspose.3D?
Экспорт FBX – это преобразование построенного в памяти графа сцены, созданного с помощью Aspose.3D, в файл FBX, который можно открыть в популярных 3D‑инструментах, таких как Blender, Maya или Unity. API берёт на себя тяжёлую работу, позволяя вам сосредоточиться на создании сцены.

## Почему нужно строить иерархии узлов перед экспортом?
Хорошо структурированная иерархия узлов позволяет применять преобразования один раз к родительскому узлу, автоматически влияя на всех его потомков. Это уменьшает дублирование кода и отражает реальные отношения объектов (например, шасси автомобиля с колесами как дочерними узлами).

## Предварительные требования

Прежде чем приступить, убедитесь, что у вас есть:

1. **Среда разработки Java** – JDK 8+ и IDE или система сборки по вашему выбору.  
2. **Библиотека Aspose.3D for Java** – скачайте и установите её со [страницы загрузки](https://releases.aspose.com/3d/java/).  
3. **Каталог документов** – папка на вашем компьютере, куда будет сохранён сгенерированный файл FBX.

## Импорт пакетов

Начните с импорта необходимых классов Aspose.3D:

```java
import com.aspose.threed.*;

```

## Шаг 1: Инициализация объекта Scene

```java
// Initialize scene object
Scene scene = new Scene();
```

## Шаг 2: Создание дочерних узлов и добавление меша к узлу

На этом этапе мы демонстрируем **как создавать дочерние узлы** и **добавлять меш к объектам узла**.

```java
// Get a child node object
Node top = scene.getRootNode().createChildNode();

// Create the first cube node
Node cube1 = top.createChildNode("cube1");
Mesh mesh = Common.createMeshUsingPolygonBuilder(); // Use your mesh creation method
cube1.setEntity(mesh);
cube1.getTransform().setTranslation(new Vector3(-10, 0, 0));

// Create the second cube node
Node cube2 = top.createChildNode("cube2");
cube2.setEntity(mesh);
cube2.getTransform().setTranslation(new Vector3(10, 0, 0));
```

## Шаг 3: Применение вращения к верхнему узлу

Вращение родительского узла автоматически вращает всех его потомков, что является ключевым преимуществом иерархических сцен.

```java
// Rotate the top node, affecting all child nodes
top.getTransform().setRotation(Quaternion.fromEulerAngle(Math.PI, 4, 0));
```

## Шаг 4: Сохранение 3D‑сцены – Как экспортировать FBX

Теперь мы **сохраняем сцену как FBX**, завершая процесс «как экспортировать FBX».

```java
// Save 3D scene in the supported file format (FBX in this case)
String MyDir = "Your Document Directory";
MyDir = MyDir + "NodeHierarchy.fbx";
scene.save(MyDir, FileFormat.FBX7500ASCII);
System.out.println("\nNode hierarchy added successfully to document.\nFile saved at " + MyDir);
```

### Ожидаемый результат
Запуск кода создаёт файл **NodeHierarchy.fbx** в указанном каталоге. Откройте его в любом FBX‑совместимом просмотрщике, чтобы увидеть два куба, расположенных слева и справа от центрального pivota, вращающихся вместе.

## Распространённые проблемы и решения

| Проблема | Почему происходит | Решение |
|----------|-------------------|---------|
| **Ошибка «File not found»** при сохранении | Неправильный путь `MyDir` или отсутствует завершающий разделитель | Убедитесь, что каталог существует и заканчивается разделителем (`/` или `\\`). |
| **Меш не виден** после экспорта | Меш не привязан к сущности или трансляция переместила его за пределы видимости | Проверьте `cube1.setEntity(mesh)` и значения трансляции. |
| **Вращение выглядит неправильно** | Неправильное использование радианов вместо градусов | `Quaternion.fromEulerAngle` ожидает радианы; скорректируйте значения. |
