# Создание 3D‑сцены в Java – применение PBR‑материалов с Aspose.3D

## Введение

В этом практическом руководстве вы узнаете, **как создать 3D‑сцену в Java** и обогатить её материалами Physically Based Rendering (PBR) с помощью библиотеки Aspose.3D. По завершении руководства вы сможете визуализировать реалистичные поверхности и **сохранить 3D‑модель в формате STL** для дальнейшего использования в любой 3D‑конвейер.

## Быстрые ответы
- **Что означает «create 3d scene java»?** Это процесс создания объекта `Scene`, который содержит геометрию, источники света и камеры в Java‑приложении.  
- **Какая библиотека работает с PBR‑материалами?** Aspose.3D предоставляет готовый класс `PbrMaterial`.  
- **Можно ли экспортировать результат в STL?** Да – метод `Scene.save` поддерживает STL (ASCII и binary).  
- **Нужна ли лицензия для продакшн‑использования?** Для коммерческого применения требуется постоянная лицензия Aspose.3D; временная лицензия подходит для тестирования.  
- **Какая версия Java требуется?** Поддерживается любой runtime Java 8+.

## Что такое 3D‑сцена в Java?

*Сцена* – это контейнер, в котором находятся все объекты (узлы), их геометрия, материалы, источники света и камеры. Представьте её как виртуальную сцену, на которой размещаются ваши 3D‑модели. Класс `Scene` из Aspose.3D предоставляет чистый объектно‑ориентированный способ программно построить эту сцену.

## Почему стоит использовать PBR‑материалы для рендеринга 3D‑объектов в Java?

PBR (Physically Based Rendering) имитирует взаимодействие света с реальными материалами, используя параметры, такие как коэффициенты металлическости и шероховатости. Результат – более правдоподобный вид при разных условиях освещения, что особенно ценно для визуализации продуктов, игр или AR/VR‑приложений.

## Импорт пакетов

Добавьте пространство имён Aspose.3D в ваш Java‑файл:

```java
import com.aspose.threed.*;
```

## Шаг 1: Инициализация сцены

Создайте сцену, которая будет служить холстом для вашей геометрии и материалов.

```java
// ExStart:InitializeScene
String MyDir = "Your Document Directory";
Scene scene = new Scene();
// ExEnd:InitializeScene
```

> **Pro tip:** Держите `MyDir` указывающим на папку с правом записи; иначе вызов `save` завершится ошибкой.

## Шаг 2: Инициализация PBR‑материала

Настройте PBR‑материал с параметрами металлическости и шероховатости, которые дают почти металлический вид.

```java
// ExStart:InitializePBRMaterial
PbrMaterial mat = new PbrMaterial();
mat.setMetallicFactor(0.9);
mat.setRoughnessFactor(0.9);
// ExEnd:InitializePBRMaterial
```

> **Почему такие значения?** Высокий коэффициент металлическости (≈ 0.9) заставляет поверхность вести себя как металл, а высокая шероховатость (≈ 0.9) добавляет лёгкое рассеивание, предотвращая идеальное зеркальное отражение.

## Шаг 3: Создание 3D‑объекта и применение материала

Здесь мы генерируем простую геометрию коробки, присоединяем её к корневому узлу сцены и назначаем только что созданный PBR‑материал.

```java
// ExStart:Create3DObject
Node boxNode = scene.getRootNode().createChildNode("box", new Box());
boxNode.setMaterial(mat);
// ExEnd:Create3DObject
```

> **Распространённая ошибка:** Если не задать материал узлу, объект останется с материалом по умолчанию (не‑PBR).

## Шаг 4: Сохранение 3D‑сцены (экспорт в STL)

Экспортируйте всю сцену – включая коробку с PBR‑материалом – в файл STL. STL – широко поддерживаемый формат для 3D‑печати и быстрых визуальных проверок.

```java
// ExStart:Save3DScene
scene.save(MyDir + "PBR_Material_Box_Out.stl", FileFormat.STLASCII);
// ExEnd:Save3DScene
```

При необходимости можно выбрать `FileFormat.STLBINARY` для уменьшения размера файла.

## Распространённые проблемы и их решения

| Проблема | Возможная причина | Решение |
|----------|-------------------|---------|
| **Файл не сохранён** | `MyDir` указывает на несуществующую папку или нет прав записи | Убедитесь, что каталог существует и процесс Java имеет право записи |
| **Материал выглядит плоским** | Значения Metallic/Roughness установлены в 0 | Увеличьте `setMetallicFactor` и/или `setRoughnessFactor` |
| **Не удаётся открыть STL‑файл** | Неправильный флаг формата файла (ASCII vs Binary) для просмотрщика | Используйте соответствующее значение перечисления `FileFormat` для вашего просмотрщика |
