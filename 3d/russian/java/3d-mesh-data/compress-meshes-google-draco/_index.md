# Сократите размер 3D‑модели: создайте сферическую сетку в Java с помощью Draco

## Введение

Если вы ищете быстрый способ **сократить размер 3D‑модели**, одновременно сохраняя высококачественную геометрию, вы попали в нужное место. В этом учебнике мы пройдем процесс создания сферической сетки с помощью **Aspose.3D for Java**, а затем сожмём её с помощью **Google Draco**. К концу вы получите готовый файл `.drc`, который будет значительно меньше оригинала, что идеально подходит для веб‑просмотрщиков, мобильных игр или любого Java‑приложения с ограниченной пропускной способностью.

## Быстрые ответы
- **Что покрывает этот учебник?** Создание сферической сетки в Java и её сжатие с помощью Google Draco через Aspose.3D.  
- **Основная библиотека?** Aspose.3D for Java (используется как для создания сетки, так и для экспорта в Draco).  
- **Типичное время реализации?** Около 10‑15 минут для базовой сферы.  
- **Ключевое требование?** Среда разработки Java с JAR‑файлами Aspose.3D в classpath.  
- **Результат?** Файл `.drc`, который **сокращает размер 3D‑модели** до 90 % по сравнению с несжатой сеткой.

## Что означает «сократить размер 3D‑модели» в контексте 3D‑разработки?

Сокращение размера 3D‑модели подразумевает уменьшение объёма геометрических данных, которые необходимо передавать или хранить, без заметного ухудшения визуального качества. Draco достигает этого, кодируя позиции вершин, нормали и другие атрибуты в сильно сжатом бинарном формате. В сочетании с Aspose.3D весь процесс остаётся в Java, без необходимости работать с нативными бинарными файлами.

## Почему использовать сжатие сетки Google Draco с Aspose.3D?

- **Огромное уменьшение размера:** Draco может сократить данные сетки до 90 % по сравнению с форматами OBJ или STL.  
- **Быстрое декодирование во время выполнения:** Движки такие как Unity, Unreal и three.js нативно декодируют Draco, ускоряя загрузку.  
- **Бесшовная интеграция с Java:** Aspose.3D абстрагирует нативную библиотеку Draco, позволяя оставаться в экосистеме Java.  
- **Единый экспорт Aspose 3D:** Один и тот же API, которым вы создаёте геометрию, также отвечает за экспорт, упрощая конвейер.

## Требования

- **Java Development Kit (JDK)** – версия 8 или новее.  
- **Aspose.3D for Java** – загрузите последние JAR‑файлы со страницы [here](https://releases.aspose.com/3d/java/).  
- **Базовое знакомство с Google Draco** – вы будете использовать обёртку Aspose.3D, поэтому установка нативного Draco не требуется.

## Импорт пакетов

```java
import com.aspose.threed.DracoCompressionLevel;
import com.aspose.threed.DracoSaveOptions;
import com.aspose.threed.FileFormat;
import com.aspose.threed.Sphere;


import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Paths;
```

## Пошаговое руководство

### Шаг 1: Настройка проекта

Создайте новый Java‑проект (подойдёт любой IDE) и добавьте все JAR‑файлы Aspose.3D в classpath. Разместите исходные файлы в пакете, например `com.example.draco`, для удобства.

### Шаг 2: Как создать сферическую сетку в Java

```java
// ExStart:Encode3DMeshinGoogleDraco
// The path to the documents directory.
String MyDir = "Your Document Directory";

// Create a sphere
Sphere sphere = new Sphere();
```

> **Pro tip:** Класс `Sphere` генерирует триангулированную сетку с радиусом по умолчанию 1.0. Вы можете задать собственный радиус, степень детализации или параметры материала, если требуется иной уровень детализации перед сжатием.

### Шаг 3: Как сжать сетку с помощью Google Draco

```java
// Encode the sphere to Google Draco raw data using optimal compression level.
DracoSaveOptions opt = new DracoSaveOptions();
opt.setCompressionLevel(DracoCompressionLevel.OPTIMAL);
byte[] b = FileFormat.DRACO.encode(sphere.toMesh(), opt);
```

Установка уровня сжатия в `OPTIMAL` обеспечивает максимальное уменьшение размера при сохранении визуального качества, напрямую помогая вам **сократить размер 3D‑модели**.

### Шаг 4: Сохранить сжатую сетку

```java
// Save the raw bytes to file
Files.write(Paths.get(MyDir, "SphereMeshtoDRC_Out.drc"), b);
// ExEnd:Encode3DMeshinGoogleDraco
```

Полученный файл `SphereMeshtoDRC_Out.drc` можно транслировать клиентам, хранить в CDN или загружать напрямую любой движок, поддерживающий Draco.

## Распространённые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| **`NoClassDefFoundError` для классов Draco** | JAR‑файлы Aspose.3D не находятся в classpath | Убедитесь, что *все* JAR‑файлы Aspose.3D включены и версия соответствует документации. |
| **Файл вывода пустой** | `MyDir` указывает на несуществующую папку | Создайте каталог программно (`Files.createDirectories(Paths.get(MyDir))`) перед записью файла. |
| **Сжатая сетка выглядит искажённой** | Используется низкий уровень сжатия или недостаточная детализация | Переключитесь на `DracoCompressionLevel.OPTIMAL` и увеличьте детализацию сферы (например, `new Sphere(1.0, 64, 64)`). |
