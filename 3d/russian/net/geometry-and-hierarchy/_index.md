# Геометрия и Иерархия

## Введение

Добро пожаловать в окончательное руководство по **geometric transformation 3d** с использованием Aspose.3D для .NET. Независимо от того, только начинаете вы или уже несколько лет создаёте 3D‑приложения, эта коллекция практических учебников поможет вам освоить всё — от применения PBR‑материалов до манипулирования узлами с помощью матрицы преобразования. **Вы узнаете, как применять PBR — конкретно, как применять pbr‑материалы к вашим 3D‑моделям**. Давайте погрузимся и посмотрим, как превратить концепции в реальные 3D‑сцены.

## Быстрые ответы
- **Что такое PBR?** Physically‑Based Rendering (PBR) моделирует реалистичный отклик материала на свет.  
- **Почему использовать Aspose.3D для PBR?** Он предоставляет простой .NET API для создания материалов и их рендеринга.  
- **Нужна ли лицензия?** Доступна бесплатная пробная версия; для продакшна требуется коммерческая лицензия.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **Можно ли комбинировать PBR с пользовательскими шейдерами?** Да, вы можете расширять материалы пользовательским кодом шейдера.

## Как применять материалы PBR в Aspose.3D
Применение PBR‑материалов является краеугольным камнем создания фотореалистичных 3D‑сцен. В этом разделе мы пройдем основные шаги — создание материала, назначение его геометрии и рендеринг результата. Этот же подход используется во многих последующих учебниках, поэтому его освоение ускорит работу со всеми другими примерами.

### Почему использовать PBR?
- **Реализм:** PBR имитирует взаимодействие света с поверхностями в реальном мире.  
- **Последовательность:** Материалы выглядят правильно при любом освещении.  
- **Производительность:** Оптимизированные шейдеры делают PBR пригодным для интерактивных приложений.

### Требования
- Aspose.3D for .NET (последняя версия) установлен.  
- Среда разработки .NET (Visual Studio 2022 или новее).

### Как это вписывается в другие темы
Как только вы узнаете **how to apply pbr**, вы сможете легко комбинировать его с другими преобразованиями, такими как **euler angle rotation**, смешивание кватернионов или прямое манипулирование матрицей — всё это покрыто в последующих учебниках.

## Часто задаваемые вопросы

**Q: Как начать новый проект, использующий PBR‑материалы?**  
A: Установите пакет Aspose.3D NuGet, создайте `Scene` и следуйте учебнику «Applying PBR Material to Box», чтобы добавить ваш первый PBR‑материал.

**Q: Можно ли использовать эти учебники с .NET Core?**  
A: Да, все примеры работают с .NET Core 3.1 и новее, а также с .NET 5/6.

**Q: Какой лучший способ выполнить запрос к иерархии узлов?**  
A: Используйте синтаксис запросов, похожий на XPath, продемонстрированный в учебнике «XPath‑Like Object Queries» — это самый лаконичный метод для **how to query hierarchy**.

**Q: Я получаю некорректное освещение на моём кубе — что проверить?**  
A: Убедитесь, что нормали заданы корректно (см. «Setting Up Normals on Cube») и что ваш PBR‑материал содержит правильные значения metallic и roughness.

**Q: Есть ли соображения по производительности при использовании большого количества матриц преобразования?**  
A: По возможности группируйте преобразования и переиспользуйте объекты матриц, чтобы уменьшить количество выделений памяти; руководство «Transforming Node by Transformation Matrix» содержит рекомендации по лучшим практикам.
