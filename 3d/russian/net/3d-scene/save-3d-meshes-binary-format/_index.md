# Сохранить 3D‑модель в пользовательском бинарном формате с помощью Aspose.3D для .NET

## Введение

Добро пожаловать! В этом **уроке Aspose 3D** вы узнаете, как **сохранить 3D‑модель** в пользовательском бинарном формате. Независимо от того, нужно ли вам **конвертировать бинарные FBX**‑файлы для игрового движка или создать собственный лёгкий контейнер моделей, это руководство проведёт вас через каждый шаг с понятными примерами на C#. Предполагается, что вы знакомы с базовым синтаксисом C# и у вас установлен Aspose.3D.

## Быстрые ответы
- **Что покрывает этот урок?** Сохранение 3D‑модели в пользовательский бинарный файл с помощью Aspose.3D для .NET.  
- **Какие форматы файлов можно конвертировать?** Любой формат, поддерживаемый Aspose.3D (например, FBX, OBJ, 3DS) – в примере используется FBX.  
- **Нужна ли лицензия?** Для разработки достаточно бесплатной пробной версии; для продакшна требуется коммерческая лицензия.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **Сколько времени занимает реализация?** Около 10‑15 минут для базовой конверсии.

## Что такое **save 3d mesh**?

Сохранение 3D‑модели означает извлечение позиций вершин, нормалей, UV‑координат и индексов треугольников из сцены и запись их в файл, определённый вами. Пользовательский бинарный формат даёт полный контроль над размером хранилища и скоростью чтения, что критично для рендеринга в реальном времени или передачи по сети.

## Почему **convert FBX binary** и **create custom mesh format**?

- **Производительность:** Бинарные данные загружаются быстрее, чем текстовые форматы вроде OBJ.  
- **Переносимость:** Пользовательский формат можно адаптировать под точные требования вашего движка, убрав лишние данные.  
- **Безопасность:** Бинарные файлы реже подвергаются случайному редактированию, что помогает защитить собственную геометрию.  

Использование Aspose.3D упрощает процесс конверсии, сохраняя код чистым и поддерживаемым.

## Шаг 1: Загрузка 3D‑файла

Загрузите ваш 3D‑файл с помощью Aspose.3D. В этом примере мы загружаем файл **test.fbx**:

```csharp
Scene scene = Scene.FromFile("test.fbx");
```

Метод `Scene.FromFile` автоматически определяет исходный формат, поэтому вы можете заменить FBX‑файл на OBJ, 3DS или любой другой поддерживаемый тип.

## Шаг 2: Определение пользовательского бинарного формата

Определите структуру пользовательского бинарного формата, в который вы будете сохранять 3D‑модели. В примере используется структура с `MeshBlock`, `Vertex` и `Triangle` в качестве компонентов.

```csharp
//The memory layout of a vertex is 
// float[3] position;
// float[3] normal;
// float[3] uv;
var vertexDeclaration = new VertexDeclaration();
vertexDeclaration.AddField(VertexFieldDataType.FVector3, VertexFieldSemantic.Position);
vertexDeclaration.AddField(VertexFieldDataType.FVector3, VertexFieldSemantic.Normal);
vertexDeclaration.AddField(VertexFieldDataType.FVector3, VertexFieldSemantic.UV);
```

Объявляя раскладку вершин, вы сообщаете Aspose.3D, как упаковывать данные перед записью в бинарный поток.

## Шаг 3: Открытие файла для записи

Откройте бинарный файл для записи, куда будут сохраняться конвертированные 3D‑модели:

```csharp
using (var writer = new BinaryWriter(new FileStream("Your Output Directory" + "Save3DMeshesInCustomBinaryFormat_out", FileMode.Create, FileAccess.Write)))
```

`BinaryWriter` даёт низкоуровневый контроль над порядком байтов и гарантирует, что файл будет создан заново при каждом запуске.

## Шаг 4: Итерация по узлам и сущностям

Пройдите каждый узел в 3D‑сцене и конвертируйте сущности сеток в пользовательский бинарный формат. Игнорируйте источники света, камеры и другие не‑сеточные сущности:

```csharp
scene.RootNode.Accept(delegate(Node node)
{
    foreach (Entity entity in node.Entities)
    {
        if (!(entity is IMeshConvertible))
            continue;
        // ... (continue processing)
    }
    return true;
});
```

Метод `Accept` выполняет обход в глубину, позволяя обработать каждую сетку независимо от уровня вложенности.

## Шаг 5: Конвертация и запись контрольных точек и треугольников

Для каждой сетки конвертируйте контрольные точки в мировое пространство и запишите их в бинарный файл вместе с индексами треугольников:

```csharp
Mesh m = ((IMeshConvertible)entity).ToMesh();

var triMesh = TriMesh.FromMesh(vertexDeclaration, m);


//The mesh's memory layout is:
// float[16] transform_matrix;
// int vertices_count;
// int indices_count;
// vertex[vertices_count] vertices;
// ushort[indices_count] indices;


//write transform
var transform = node.GlobalTransform.TransformMatrix.ToArray();
for(int i = 0; i < transform.Length; i++)
    writer.Write((float)transform[i]);
//write number of vertices/indices
writer.Write(triMesh.VerticesCount);
writer.Write(triMesh.IndicesCount);
//write vertices and indices
writer.Flush();
triMesh.WriteVerticesTo(writer.BaseStream);
triMesh.Write16bIndicesTo(writer.BaseStream);
```

Этот блок сначала записывает матрицу трансформации узла в мировом пространстве, затем количество вершин, количество индексов, буфер вершин и, наконец, 16‑битный буфер индексов. Полученный файл можно эффективно читать любой движок, знающий эту раскладку.

## Распространённые проблемы и решения

- **Ошибки пути к файлу:** Убедитесь, что выходной каталог существует, либо используйте `Path.Combine` для построения корректного пути.  
- **Большие модели:** Для моделей с миллионами вершин рассматривайте запись частями, чтобы избежать `OutOfMemoryException`.  
- **Несоответствия систем координат:** Aspose.3D использует правостороннюю систему координат; при необходимости преобразуйте её в левостороннюю для вашего целевого движка.

## Часто задаваемые вопросы

**Вопрос:** Работает ли этот подход с анимированными моделями?  
**Ответ:** Да, вы можете экспортировать вершины каждого кадра, проходя по ключевым кадрам анимации перед записью бинарных данных.

**Вопрос:** Можно ли добавить пользовательские атрибуты вершин, такие как веса костей?  
**Ответ:** Конечно. Расширьте `VertexDeclaration`, добавив дополнительные поля (например, `VertexFieldSemantic.BoneWeight`) и запишите дополнительные данные после стандартного блока вершин.

**Вопрос:** Как лучше всего читать пользовательский бинарный файл обратно в сцену?  
**Ответ:** Реализуйте соответствующий бинарный читатель, который считывает матрицу трансформации, количество вершин, количество индексов, а затем восстанавливает `TriMesh` с помощью `TriMesh.FromBinary`.
