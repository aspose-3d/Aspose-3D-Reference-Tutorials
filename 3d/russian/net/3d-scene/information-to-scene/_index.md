# Как добавить информацию о поставщике и сохранить сцену FBX с помощью Aspose.3D

## Введение

Добро пожаловать в этот подробный учебник, который показывает **как добавить** сведения о поставщике в 3D‑сцену и затем **как сохранить** файлы FBX с помощью Aspose.3D для .NET. Независимо от того, создаёте ли вы архитектурные визуализации, игровые ассеты или инженерные модели, внедрение метаданных о поставщике и приложении делает ваши сцены более информативными и упрощает их дальнейшее управление. Давайте пройдём процесс шаг за шагом.

## Быстрые ответы
- **Что значит «добавить поставщика»?** Это сохраняет названия приложения и поставщика внутри блока AssetInfo сцены.  
- **Какой формат поддерживает информацию о поставщике?** FBX (ASCII или бинарный) сохраняет эти метаданные при сохранении.  
- **Как сохранить FBX?** Используйте `scene.Save(path, FileFormat.FBX7500ASCII)` или эквивалентный бинарный вариант.  
- **Нужна ли лицензия?** Для разработки подходит бесплатная пробная версия; для продакшна требуется коммерческая лицензия.  
- **Можно ли изменить единицы измерения?** Да, задайте `AssetInfo.UnitName` и `AssetInfo.UnitScaleFactor`.

## Что означает «как добавить поставщика» в 3D‑сцене?
Добавление информации о поставщике означает заполнение свойств `AssetInfo` объекта `Scene`. Эти свойства сохраняются в файле, позволяя любому получателю FBX‑файла увидеть, какое приложение его создало и кто является поставщиком.

## Почему стоит добавлять информацию о поставщике?
- **Отслеживаемость:** Быстро определить источник модели в больших конвейерах.  
- **Соответствие требованиям:** В некоторых отраслях требуется явное указание поставщика для управления активами.  
- **Автоматизация:** Скрипты могут фильтровать или обрабатывать файлы на основе метаданных поставщика.

## Предварительные требования

- Aspose.3D для .NET установлен. Вы можете скачать его со страницы [Aspose.3D for .NET](https://releases.aspose.com/3d/net/).

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
```

## Как добавить информацию о поставщике

### Шаг 1: Инициализировать 3D‑сцену

```csharp
Scene scene = new Scene();
```

Создание новой `Scene` даёт вам чистый холст для работы.

### Шаг 2: Установить информацию о приложении и поставщике

```csharp
scene.AssetInfo.ApplicationName = "Egypt";
scene.AssetInfo.ApplicationVendor = "Manualdesk";
```

Здесь мы демонстрируем **как добавить поставщика**, присваивая осмысленные строки свойствам `ApplicationName` и `ApplicationVendor`.

### Шаг 3: Определить единицы измерения

```csharp
scene.AssetInfo.UnitName = "pole";
scene.AssetInfo.UnitScaleFactor = 0.6;
```

Указание системы единиц гарантирует, что любой, открывающий FBX‑файл, правильно интерпретирует размеры. В этом примере один «столб» равен 60 см.

## Как сохранить сцену FBX

### Шаг 4: Сохранить сцену (как сохранить fbx)

```csharp
var output = "Your Output Directory" + "InformationToScene.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

Эта строка показывает **как сохранить fbx** с использованием ASCII‑версии FBX 7.5.0. Если предпочитаете бинарный формат, замените `FBX7500ASCII` на `FBX7500Binary`.

> **Полезный совет:** Сохраняйте расширение файла `.fbx` в соответствии с выбранным форматом; иначе некоторые просмотрщики могут неверно интерпретировать содержимое.

### Шаг 5: Вывести сообщение об успехе

```csharp
Console.WriteLine("\nAsset information added successfully to Scene.\nFile saved at " + output);
```

Дружественное сообщение в консоли подтверждает, что сцена, включающая метаданные поставщика, записана на диск.

## Распространённые проблемы и решения

| Проблема | Решение |
|----------|---------|
| **Информация о поставщике не отображается в просмотрщике** | Убедитесь, что файл сохранён как **FBX ASCII** или **Binary**; некоторые старые просмотрщики читают только один из форматов. |
| **Путь содержит пробелы** | Оберните путь в кавычки или используйте `Path.Combine` для построения безопасного пути. |
| **Масштаб единиц выглядит неверно** | Проверьте `UnitScaleFactor`; это множитель относительно метров. |
| **Исключение лицензии** | Используйте бесплатную пробную версию для тестов; получите полную лицензию для продакшн‑сборок. |
