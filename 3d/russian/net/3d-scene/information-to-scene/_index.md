# Как добавить метаданные: извлечение информации в активы сцены

## Введение

В этом всестороннем руководстве вы узнаете **как добавить метаданные** к вашим 3D‑сценам с помощью Aspose.3D for .NET. Добавление метаданных, таких как сведения о поставщике, пользовательские единицы измерения и другая информация об активе, делает ваши модели более информативными, удобными для поиска и готовыми к дальнейшим конвейерам, например, игровым движкам или платформам AR/VR.

## Быстрые ответы
- **Какова основная цель?** Встроить метаданные (информация о поставщике, единицы, пользовательские теги) непосредственно в 3D‑сцену.  
- **Какая библиотека используется?** Aspose.3D for .NET.  
- **Могу ли я экспортировать модель после добавления метаданных?** Да — вы можете **export 3D model** файлы, например, сохранить как FBX.  
- **Нужна ли лицензия?** Доступна бесплатная пробная версия; для продакшна требуется коммерческая лицензия.  
- **Какие версии .NET поддерживаются?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6.

## Что означает «как добавить метаданные» в 3D‑сцене?

Добавление метаданных означает привязку описательной информации — такой как название приложения, поставщик, единицы измерения или пользовательские теги — к самому файлу сцены. Эти данные перемещаются вместе с моделью, когда вы **save scene as FBX** или в другие поддерживаемые форматы, позволяя downstream‑инструментам понимать контекст без внешних файлов.

## Зачем встраивать пользовательские метаданные и информацию о поставщике?

- **Поисковость:** Системы управления активами могут фильтровать модели по поставщику или типу единиц.  
- **Совместимость:** Движки, читающие FBX, могут автоматически применять правильный масштаб, если вы **define measurement units**.  
- **Брендинг:** Включение названия приложения и поставщика усиливает владение и соблюдение лицензий.  

## Требования

Прежде чем мы начнём, убедитесь, что у вас есть:

- Aspose.3D for .NET установлен. Вы можете скачать его со страницы [Aspose.3D for .NET page](https://releases.aspose.com/3d/net/).

## Импорт пространств имён

```csharp
using System;
using System.IO;

using Aspose.ThreeD;
```

## Шаг 1: Инициализация 3D‑сцены

```csharp
Scene scene = new Scene();
```

Создайте новый объект `Scene`, который будет содержать всю геометрию и информацию об активе.

## Шаг 2: Установить приложение и **добавить информацию о поставщике**

```csharp
scene.AssetInfo.ApplicationName = "Egypt";
scene.AssetInfo.ApplicationVendor = "Manualdesk";
```

Здесь мы встраиваем **название приложения** и **информацию о поставщике**. Это ключевая часть **how to add metadata**, которая идентифицирует источник модели.

## Шаг 3: **Определить единицы измерения** для точного масштабирования

```csharp
scene.AssetInfo.UnitName = "pole";
scene.AssetInfo.UnitScaleFactor = 0.6;
```

Указывая `UnitName` и `UnitScaleFactor`, вы сообщаете downstream‑инструментам, что один «столб» равен 0,6 метра (60 см). Этот шаг необходим, когда вы позже **export 3D model** файлы, чтобы обеспечить правильные реальные размеры.

## Шаг 4: **Сохранить сцену как FBX** – **Экспортировать 3D‑модель** с метаданными

```csharp
var output = "Your Output Directory" + "InformationToScene.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

Вызов `Save` записывает сцену в файл FBX, встраивая все добавленные метаданные. Это демонстрирует **how to add metadata**, а затем **save scene as FBX**, эффективно **export 3D model** с полной информацией об активе.

## Шаг 5: Показать сообщение об успехе

```csharp
Console.WriteLine("\nAsset information added successfully to Scene.\nFile saved at " + output);
```

Дружественное сообщение в консоли подтверждает, что метаданные записаны, и указывает расположение файла.

## Распространённые проблемы и советы

- **Неправильный масштаб единицы:** Дважды проверьте, что `UnitScaleFactor` соответствует реальному преобразованию; иначе экспортированные модели могут выглядеть слишком большими или маленькими.  
- **Отсутствует информация о поставщике:** Если `ApplicationVendor` оставлен пустым, некоторые просмотрщики покажут «Unknown». Старайтесь задавать его, когда это возможно.  
- **Ошибки пути к файлу:** Убедитесь, что выходной каталог существует; иначе `scene.Save` выбросит исключение.
