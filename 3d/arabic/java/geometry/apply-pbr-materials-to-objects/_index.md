# إنشاء مشهد ثلاثي الأبعاد في Java – تطبيق مواد PBR باستخدام Aspose.3D

## المقدمة

في هذا الدرس العملي ستتعلم **كيفية إنشاء مشهد ثلاثي الأبعاد في Java** وإثرائه بمواد Physically Based Rendering (PBR) باستخدام مكتبة Aspose.3D. في نهاية الدليل ستتمكن من عرض أسطح واقعية و**حفظ نموذج 3D بصيغة STL** لاستخدامه لاحقًا في أي خط أنابيب ثلاثي الأبعاد.

## إجابات سريعة
- **ماذا يعني “إنشاء مشهد ثلاثي الأبعاد Java”؟** إنه عملية بناء كائن Scene يحتوي على الهندسة، الأضواء، والكاميرات في تطبيق Java.  
- **أي مكتبة تتعامل مع مواد PBR؟** توفر Aspose.3D فئة `PbrMaterial` جاهزة.  
- **هل يمكنني تصدير النتيجة بصيغة STL؟** نعم – تدعم طريقة `Scene.save` صيغة STL (ASCII وbinary).  
- **هل أحتاج إلى ترخيص للاستخدام التجاري؟** يتطلب الاستخدام التجاري ترخيص دائم لـ Aspose.3D؛ الترخيص المؤقت يكفي للاختبار.  
- **ما نسخة Java المطلوبة؟** أي بيئة تشغيل Java 8+ مدعومة.

## ما هو المشهد ثلاثي الأبعاد في Java؟

*المشهد* هو الحاوية التي تحتفظ بجميع الكائنات (العُقد)، هندستها، موادها، أضوائها، وكاميراتها. فكر فيه كأنه مسرح افتراضي تضع فيه نماذجك الثلاثية الأبعاد. توفر فئة `Scene` في Aspose.3D طريقة نظيفة كائنية‑التوجه لبناء هذا المسرح برمجيًا.

## لماذا نستخدم مواد PBR لتصيير الكائنات الثلاثية الأبعاد في Java؟

PBR (Physically Based Rendering) يحاكي تفاعل الضوء في العالم الحقيقي باستخدام معلمات مثل عامل المعدن والخشونة. النتيجة هي مظهر أكثر إقناعًا عبر ظروف إضاءة مختلفة، وهو أمر ذو قيمة خاصة لتصوير المنتجات، الألعاب، أو تجارب AR/VR.

## استيراد الحزم

أضف مساحة أسماء Aspose.3D إلى ملف مصدر Java الخاص بك:

```java
import com.aspose.threed.*;
```

## الخطوة 1: تهيئة مشهد

أنشئ المشهد الذي سيعمل كقماش لهندستك وموادك.

```java
// ExStart:InitializeScene
String MyDir = "Your Document Directory";
Scene scene = new Scene();
// ExEnd:InitializeScene
```

> **نصيحة احترافية:** احرص على أن يشير `MyDir` إلى مجلد قابل للكتابة؛ وإلا سيفشل استدعاء `save`.

## الخطوة 2: تهيئة مادة PBR

قم بتكوين مادة PBR بقيم معدنية وخشونة تعطي مظهرًا شبه معدني.

```java
// ExStart:InitializePBRMaterial
PbrMaterial mat = new PbrMaterial();
mat.setMetallicFactor(0.9);
mat.setRoughnessFactor(0.9);
// ExEnd:InitializePBRMaterial
```

> **لماذا هذه القيم؟** عامل معدني عالي (≈ 0.9) يجعل السطح يتصرف كالمعدن، بينما خشونة عالية (≈ 0.9) تضيف انتشارًا طفيفًا، مما يمنع الانعكاس المثالي كالمرآة.

## الخطوة 3: إنشاء كائن ثلاثي الأبعاد وتطبيق المادة

هنا نقوم بإنشاء هندسة صندوق بسيط، ربطه بعقدة الجذر في المشهد، وتعيين مادة PBR التي أنشأناها للتو.

```java
// ExStart:Create3DObject
Node boxNode = scene.getRootNode().createChildNode("box", new Box());
boxNode.setMaterial(mat);
// ExEnd:Create3DObject
```

> **خطأ شائع:** نسيان تعيين المادة على العقدة سيترك الكائن بالمظهر الافتراضي (غير PBR).

## الخطوة 4: حفظ المشهد ثلاثي الأبعاد (تصدير STL)

صدّر المشهد بالكامل—بما في ذلك الصندوق المعزز بـ PBR—إلى ملف STL. STL هو تنسيق مدعوم على نطاق واسع للطباعة ثلاثية الأبعاد وفحوصات العرض السريعة.

```java
// ExStart:Save3DScene
scene.save(MyDir + "PBR_Material_Box_Out.stl", FileFormat.STLASCII);
// ExEnd:Save3DScene
```

يمكنك أيضًا اختيار `FileFormat.STLBINARY` إذا كان حجم الملف الأصغر مطلوبًا.

## المشكلات الشائعة والحلول

| المشكلة | السبب المحتمل | الحل |
|-------|--------------|-----|
| **الملف غير محفوظ** | `MyDir` يشير إلى مجلد غير موجود أو يفتقر إلى صلاحية كتابة | تحقق من وجود الدليل وأن عملية Java لديها صلاحية كتابة |
| **المادة تظهر مسطحة** | قيم Metallic/Roughness مضبوطة على 0 | زد من `setMetallicFactor` و/أو `setRoughnessFactor` |
| **لا يمكن فتح ملف STL** | علم تنسيق الملف غير صحيح (ASCII مقابل Binary) للعارض | استخدم قيمة `FileFormat` المطابقة للعارض المستهدف |
