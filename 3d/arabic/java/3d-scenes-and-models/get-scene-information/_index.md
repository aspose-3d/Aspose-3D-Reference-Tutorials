# كيفية حفظ ملف FBX واسترجاع معلومات المشهد ثلاثي الأبعاد في Java

## مقدمة

إذا كنت تبحث عن دليل واضح وتطبيقي حول **كيفية حفظ ملفات fbx** مع استخراج البيانات الوصفية المفيدة من مشاهدك ثلاثية الأبعاد، فقد وصلت إلى المكان الصحيح. في هذا البرنامج التعليمي سنستعرض كل خطوة باستخدام مكتبة **Aspose.3D Java**: من إنشاء المشهد، **تحديد اسم التطبيق**، **تعريف وحدات القياس**، إلى **تصدير المشهد إلى FBX** في النهاية. بحلول ذلك الوقت ستحصل على ملف FBX جاهز للاستخدام يحمل معلومات الأصول التي تحتاجها لسلاسل المعالجة اللاحقة.

## إجابات سريعة
- **ما هو الهدف الأساسي؟** حفظ ملف FBX يحتوي على معلومات أصول مخصصة.  
- **ما المكتبة المستخدمة؟** Aspose.3D for Java.  
- **هل أحتاج إلى ترخيص؟** نسخة تجريبية مجانية تكفي للتطوير؛ يلزم ترخيص تجاري للإنتاج.  
- **هل يمكنني تغيير وحدات القياس؟** نعم – استخدم `setUnitName` و `setUnitScaleFactor`.  
- **أين يتم حفظ الناتج؟** في المسار الذي تحدده في `scene.save(...)`.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من وجود ما يلي:

- فهم قوي لأساسيات لغة Java.  
- **Aspose.3D for Java** تم تحميله وإضافته إلى مشروعك (يمكنك الحصول عليه من صفحة [تحميل Aspose 3D الرسمية](https://releases.aspose.com/3d/java/)).  
- بيئة تطوير Java المفضلة لديك (IntelliJ IDEA، Eclipse، NetBeans، إلخ) مُعدّة بشكل صحيح.

## استيراد الحزم

في ملف المصدر Java الخاص بك، استورد فئات Aspose.3D التي توفر معالجة المشهد ودعم صيغ الملفات.

```java
import com.aspose.threed.FileFormat;
import com.aspose.threed.Scene;
```

> **نصيحة احترافية:** حافظ على قائمة الاستيراد بأقل ما يمكن لتجنب الاعتمادات غير الضرورية وتحسين أوقات التجميع.

## ما هي عملية حفظ ملف FBX؟

فيما يلي دليل مختصر خطوة بخطوة يوضح **كيفية إضافة معلومات الأصول** إلى المشهد ثم **تصديره إلى FBX**.

### الخطوة 1: تهيئة مشهد ثلاثي الأبعاد

أولاً، أنشئ كائن `Scene` فارغ. سيكون هذا الحاوية لجميع الهندسات، الأضواء، الكاميرات، والبيانات الوصفية للأصول.

```java
// ExStart:AddAssetInformationToScene
Scene scene = new Scene();
```

### الخطوة 2: تعيين معلومات التطبيق والبائع

إضافة بيانات وصفية مخصصة تساعد الأدوات اللاحقة على تحديد مصدر الملف. هنا نقوم **بتعيين اسم التطبيق** والبائع باستخدام كائن `AssetInfo`.

```java
scene.getAssetInfo().setApplicationName("Egypt");
scene.getAssetInfo().setApplicationVendor("Manualdesk");
```

> **لماذا هذا مهم:** العديد من سلاسل المعالجة تقوم بفلترة أو وضع علامات على الأصول بناءً على التطبيق الأصلي، مما يجعل هذه الخطوة أساسية للمشاريع الكبيرة.

### الخطوة 3: تعريف وحدات القياس

تتيح لك Aspose.3D تحديد نظام الوحدات الذي يستخدمه المشهد. في هذا المثال نستخدم وحدة مصرية قديمة تُدعى “pole” مع عامل مقياس مخصص.

```java
scene.getAssetInfo().setUnitName("pole");
scene.getAssetInfo().setUnitScaleFactor(0.6);
```

> **نصيحة:** اضبط `unitScaleFactor` ليتطابق مع الحجم الحقيقي لنماذجك؛ 1.0 يمثل مطابقة 1‑ إلى‑1 مع الوحدة المختارة.

### الخطوة 4: تصدير المشهد إلى FBX

الآن بعد إرفاق معلومات الأصول، نقوم بحفظ المشهد كملف FBX. خيار `FileFormat.FBX7500ASCII` ينتج ملف FBX نصي قابل للقراءة البشرية، وهو مفيد للتصحيح.

```java
String MyDir = "Your Document Directory";
MyDir = MyDir + "InformationToScene.fbx";
scene.save(MyDir, FileFormat.FBX7500ASCII);
// ExEnd:AddAssetInformationToScene
```

> **تذكر:** استبدل `"Your Document Directory"` بمسار مطلق أو مسار نسبي إلى دليل عمل مشروعك.

### الخطوة 5: طباعة رسالة النجاح

إخراج بسيط إلى وحدة التحكم يؤكد نجاح العملية ويخبرك بمكان كتابة الملف.

```java
System.out.println("\nAsset information added successfully to Scene.\nFile saved at " + MyDir);
```

## حالات الاستخدام الشائعة

- **سلاسل أصول الألعاب** – تضمين معلومات المنشئ مباشرة في ملفات FBX لتتبع الإصدارات.  
- **التصوير المعماري** – تخزين وحدات المشروع لتجنب أخطاء التحجيم عند الاستيراد إلى محركات العرض.  
- **التقارير الآلية** – إنشاء ملفات FBX في الوقت الفعلي مع بيانات وصفية يمكن لأدوات التحليل اللاحقة قراءتها.

## استكشاف الأخطاء وإصلاحها & نصائح

| المشكلة | الحل |
|-------|----------|
| **الملف غير موجود بعد الحفظ** | تحقق من أن `MyDir` يشير إلى مجلد موجود وأن تطبيقك يمتلك صلاحيات الكتابة. |
| **الوحدات تظهر غير صحيحة في عارض خارجي** | أعد فحص `unitScaleFactor`؛ بعض العارضات تتوقع الأمتار كوحدة أساسية. |
| **بيانات الأصول مفقودة** | تأكد من استدعاء `scene.getAssetInfo()` **قبل** الحفظ؛ التغييرات التي تجرى بعد `save()` لن تُحفظ. |

## أسئلة شائعة أخرى

**س: كيف أغيّر صيغة الإخراج إلى FBX ثنائي؟**  
ج: استبدل `FileFormat.FBX7500ASCII` بـ `FileFormat.FBX7500` عند استدعاء `scene.save(...)`.

**س: هل يمكنني إضافة بيانات وصفية مخصصة من قبل المستخدم تتجاوز الحقول المدمجة للأصول؟**  
ج: نعم، استخدم `scene.getUserData().add("Key", "Value")` لتضمين أزواج مفتاح‑قيمة إضافية.

**س: هل يدعم Aspose.3D صيغ تصدير أخرى مثل OBJ أو GLTF؟**  
ج: نعم. فقط غيّر قيمة تعداد `FileFormat` إلى `OBJ` أو `GLTF2` حسب الحاجة.

**س: ما نسخة Java المطلوبة؟**  
ج: يدعم Aspose.3D for Java Java 8 وما فوق.

ج: بالطبع. حمّل الملف باستخدام `new Scene("input.fbx")`، عدّل `scene.getAssetInfo()`، ثم احفظه.
