# كيفية تقسيم الشبكة حسب المادة في جافا باستخدام Aspose.3D

## مقدمة

إذا كنت تعمل مع الرسومات ثلاثية الأبعاد في جافا، ستكتشف سريعًا أن معالجة الشبكات الكبيرة يمكن أن تصبح عنق زجاجة في الأداء—خاصةً عندما تحتوي شبكة واحدة على مواد متعددة. **تعلم كيفية تقسيم الشبكة** حسب المادة يتيح لك عزل كل مجموعة من المضلعات الخاصة بمادة معينة، مما يتيح عرضًا أسرع، وإزالة كائنات غير مرئية بسهولة، وتحكمًا أكثر دقة في معالجة القوام. في هذا البرنامج التعليمي سنستعرض الخطوات الدقيقة لـ **تقسيم الشبكة حسب المادة** باستخدام مكتبة Aspose.3D، مع شروحات عملية، ونصائح لتقليل استدعاءات الرسم، ونصائح لتحسين أداء العرض.

## إجابات سريعة
- **ماذا يعني “split mesh by material”?** يقسم شبكة واحدة إلى عدة شبكات فرعية، كل منها يحتوي على مضلعات تشترك في نفس المادة.  
- **لماذا نستخدم Aspose.3D؟** توفر API عالية المستوى ومتعددة المنصات تُجرد صيغ الملفات منخفضة المستوى مع الحفاظ على الأداء.  
- **كم من الوقت تستغرق التنفيذ؟** حوالي 10–15 دقيقة من البرمجة والاختبار.  
- **هل أحتاج إلى ترخيص؟** يتوفر إصدار تجريبي مجاني؛ يلزم الحصول على ترخيص تجاري للاستخدام في الإنتاج.  
- **ما نسخة جافا المدعومة؟** Java 8 أو أعلى.

## كيفية تقسيم الشبكة حسب المادة – نظرة عامة
تقسيم الشبكة حسب المادة هو عملية من خطوتين أساسيتين: أولاً تقوم بوسم كل مضلع بمؤشر مادة، ثم تطلب من Aspose.3D فصل الشبكة بناءً على تلك المؤشرات. النتيجة هي مجموعة من الشبكات الأصغر، يمكن لكل منها أن تُعرض باستدعاء رسم واحد—مفيد لتقليل استدعاءات الرسم وتحسين أداء العرض على وحدات معالجة الرسوميات المكتبية والمحمولة.

## ما هو تقسيم الشبكة؟

تقسيم الشبكة هو عملية تقسيم شبكة ثلاثية الأبعاد معقدة إلى قطع أصغر وأكثر قابلية للإدارة. عندما يكون التقسيم مبنيًا على المادة، يحتوي كل شبكة فرعية ناتجة على المضلعات التي تشير إلى مادة واحدة فقط. هذا النهج يقلل من استدعاءات الرسم، ويحسن أداء العرض، ويسهل تطبيق الظلال الخاصة بكل مادة.

## لماذا تقسيم الشبكة حسب المادة؟

- **الأداء:** يمكن لمحركات العرض تجميع استدعاءات الرسم per material، مما يقلل من تغييرات حالة GPU.  
- **المرونة:** يمكنك تطبيق تأثيرات ما بعد المعالجة أو منطق الاصطدام المختلف لكل مادة.  
- **إدارة الذاكرة:** الشبكات الأصغر أسهل في التدفق داخل وخارج الذاكرة، وهو أمر حاسم لتطبيقات الهواتف المحمولة أو الواقع الافتراضي.  
- **تقليل استدعاءات الرسم:** عدد أقل من تغييرات الحالة يعني أن GPU يمكنه معالجة الإطارات بشكل أكثر كفاءة.  
- **تحسين أداء العرض:** عزل المواد غالبًا ما يؤدي إلى تحسين عمليات القطع والظل.

## المتطلبات المسبقة

- معرفة أساسية ببرمجة جافا.  
- مكتبة Aspose.3D لجافا مثبتة (قم بالتنزيل من [موقع Aspose](https://releases.aspose.com/3d/java/)).  
- بيئة تطوير متكاملة مثل IntelliJ IDEA أو Eclipse أو VS Code مُعدة لتطوير جافا.

أولاً، استورد الفئات المطلوبة من Aspose.3D وأي أدوات جافا قياسية قد تحتاجها:

```java
import com.aspose.threed.*;

import java.util.Arrays;
```

## دليل خطوة بخطوة

فيما يلي استعراض مختصر لكل خطوة، مع شروحات قبل كتل الشيفرة لتعرف ما يحدث بالضبط.

### الخطوة 1: إنشاء شبكة لصندوق

نبدأ ببدائية هندسية بسيطة—صندوق—حتى نتمكن من رؤية كيف يحصل كل وجه (مستوى) على مادته الخاصة لاحقًا.

```java
// ExStart:SplitMeshbyMaterial

// Create a mesh of a box (composed of 6 planes)
Mesh box = (new Box()).toMesh();
```

### الخطوة 2: إنشاء عنصر مادة

`VertexElementMaterial` يخزن مؤشرات المواد لكل مضلع. بربطه بالشبكة، يمكننا التحكم في المادة التي يستخدمها كل وجه.

```java
// Create a material element on the box mesh
VertexElementMaterial mat = (VertexElementMaterial) box.createElement(VertexElementType.MATERIAL, MappingMode.POLYGON, ReferenceMode.INDEX);
```

### الخطوة 3: تحديد مؤشرات مواد مختلفة

هنا نُعيّن مؤشر مادة فريد لكل من المستويات الستة للصندوق. ترتيب المصفوفة يطابق ترتيب المضلعات التي يولدها الشكل `Box`.

```java
// Specify different material indices for each plane
mat.setIndices(new int[]{0, 1, 2, 3, 4, 5});
```

### الخطوة 4: تقسيم الشبكة إلى شبكات فرعية

استدعاء `PolygonModifier.splitMesh` مع `SplitMeshPolicy.CLONE_DATA` ينشئ شبكة فرعية جديدة لكل مؤشر مادة مميز مع الحفاظ على بيانات الرؤوس الأصلية.

```java
// Split the mesh into 6 sub-meshes, each plane becoming a sub-mesh
Mesh[] planes = PolygonModifier.splitMesh(box, SplitMeshPolicy.CLONE_DATA);
```

### الخطوة 5: تحديث مؤشرات المواد وإعادة التقسيم

لإظهار استراتيجية تقسيم مختلفة، نجمع الآن المستويات الثلاثة الأولى تحت مادة 0 والبقية تحت مادة 1، ثم نقسم باستخدام `COMPACT_DATA` لإزالة الرؤوس غير المستخدمة.

```java
// Update material indices and split into 2 sub-meshes
mat.getIndices().clear();
mat.setIndices(new int[]{0, 0, 0, 1, 1, 1});
planes = PolygonModifier.splitMesh(box, SplitMeshPolicy.COMPACT_DATA);
```

### الخطوة 6: تأكيد النجاح

رسالة بسيطة في وحدة التحكم تخبرك بأن العملية انتهت دون أخطاء.

```java
// Display success message
System.out.println("\nSplitting a mesh by specifying the material successfully.");
// ExEnd:SplitMeshbyMaterial
```

## تقليل استدعاءات الرسم وتحسين أداء العرض

بتحويل كل مادة إلى شبكة خاصة بها، تمكّن خط أنابيب الرسوميات من إصدار استدعاء رسم واحد لكل مادة بدلاً من استدعاء لكل مضلع. هذا التخفيض في استدعاءات الرسم يترجم مباشرة إلى إطارات أكثر سلاسة، خاصةً على الأجهزة ذات المواصفات المنخفضة. بالإضافة إلى ذلك، سياسة `COMPACT_DATA` تزيل الرؤوس غير المستخدمة، مما يقلل من عرض النطاق الترددي للذاكرة ويساعد GPU على العرض بكفاءة أكبر.

## المشكلات الشائعة والحلول

| المشكلة | سبب حدوثها | الحل |
|-------|----------------|-----|
| **الشبكات الفرعية تحتوي على رؤوس مكررة** | استخدام `CLONE_DATA` ينسخ جميع بيانات الرؤوس لكل شبكة فرعية. | التبديل إلى `COMPACT_DATA` عندما تريد إزالة التكرار من الرؤوس المشتركة. |
| **تعيين مادة غير صحيح** | طول مصفوفة المؤشرات لا يتطابق مع عدد المضلعات. | تحقق من عدد المضلعات (مثلاً، الصندوق يحتوي على 6) وقدم مصفوفة مؤشرات مطابقة. |
