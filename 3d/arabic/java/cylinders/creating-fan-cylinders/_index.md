# إضافة عقدة فرعية لبناء أسطوانات مروحة باستخدام Aspose.3D للغة Java

## المقدمة

هل أنت مستعد **لإضافة عقدة فرعية** إلى مشهد ثلاثي الأبعاد وإنشاء أسطوانات مروحة جذابة؟ في هذا الدرس سنستعرض كل خطوة — من إعداد المشهد، وضع الكائنات ثلاثية الأبعاد، وحتى **حفظ المشهد كملف OBJ** — باستخدام Aspose.3D للغة Java. سواءً كنت تُصقل أصل لعبة أو تبني نموذجًا أوليًا سريعًا، فإن المفاهيم هنا ستمنحك سيطرة قوية على نماذجك ثلاثية الأبعاد.

## إجابات سريعة
- **ماذا يفعل “إضافة عقدة فرعية”؟** يضيف كائنًا جديدًا إلى رسم بياني المشهد، مع وراثة التحولات من العنصر الأب.  
- **كيف يمكنني وضع كائن ثلاثي الأبعاد؟** عن طريق تطبيق إزاحة (أو دوران/تحجيم) على تحويل العقدة.  
- **ما هو تنسيق الملف المستخدم للتصدير؟** المثال يحفظ النموذج كملف Wavefront OBJ.  
- **هل أحتاج إلى ترخيص لتشغيل الكود؟** النسخة التجريبية المجانية تكفي للتقييم؛ الترخيص مطلوب للإنتاج.  
- **ما هو أفضل بيئة تطوير متكاملة (IDE)؟** أي IDE للغة Java (IntelliJ IDEA، Eclipse، VS Code) يدعم JDK 8+.

## ما هو “إضافة عقدة فرعية” في Aspose.3D؟
إضافة عقدة فرعية تعني إنشاء عقدة جديدة تحت عقدة أب موجودة في هيكل المشهد. الوريثة تجعل العقدة الفرعية تتبع نظام إحداثيات العنصر الأب، مما يسهل **وضع كائنات ثلاثية الأبعاد** بالنسبة لبعضها البعض.

## لماذا نضيف عقدة فرعية عند بناء أسطوانات مروحة؟
- **تصميم معياري:** كل أسطوانة (مروحة أو غير مروحة) تعيش في عقدتها الخاصة، مما يبسط التعديلات لاحقًا.  
- **وراثة التحولات:** تحريك، دوران أو تحجيم العنصر الأب ينعكس تلقائيًا على جميع الأطفال.  
- **رسم بياني للمشهد أنظف:** يحسن من قابلية القراءة وتصحيح الأخطاء للنماذج المعقدة.

## المتطلبات المسبقة

- **مجموعة تطوير جافا (JDK)** – حمّلها من [الموقع الرسمي](https://www.oracle.com/java/technologies/javase-downloads.html).  
- **Aspose.3D للغة Java** – احصل على أحدث مكتبة من [رابط التحميل](https://releases.aspose.com/3d/java/).

## استيراد الحزم

ابدأ باستيراد الحزم اللازمة إلى مشروع Java الخاص بك. هذه الخطوة أساسية للوصول إلى الوظائف التي توفرها Aspose.3D.

```java
import com.aspose.threed.*;


import java.io.IOException;
```

## الخطوة 1: إنشاء مشهد

أولاً، ننشئ مشهدًا ثلاثيًا أبعاد فارغًا سيستضيف جميع كائناتنا.

```java
// ExStart:2
// Create a Scene
Scene scene = new Scene();
// ExEnd:2
```

## الخطوة 2: إنشاء أسطوانة مروحة

بعد ذلك، نبني أسطوانة ستُعرض كمروحة (قوس جزئي).

```java
// ExStart:3
// Create a cylinder with fan
Cylinder fan = new Cylinder(2, 2, 10, 20, 1, false);
fan.setGenerateFanCylinder(true);
fan.setThetaLength(MathUtils.toRadian(270.0));
// ExEnd:3
```

## الخطوة 3: إضافة عقدة فرعية وتحديد موضع كائن ثلاثي الأبعاد

الآن **نضيف عقدة فرعية** إلى المشهد و**نحدد موضع الكائن ثلاثي الأبعاد** عبر ضبط إزاحته. هنا تصبح أسطوانة المروحة جزءًا من رسم بياني المشهد.

```java
// ExStart:4
// Create ChildNode and set translation
scene.getRootNode().createChildNode(fan).getTransform().setTranslation(10, 0, 0);
// ExEnd:4
```

## الخطوة 4: إنشاء أسطوانة غير مروحة

لإظهار مرونة Aspose.3D، ننشئ أيضًا أسطوانة عادية بدون مروحة ونضيفها كعقدة فرعية أخرى.

```java
// ExStart:5
// Create a cylinder without a fan
Cylinder nonfan = new Cylinder(2, 2, 10, 20, 1, false);
// Create ChildNode
scene.getRootNode().createChildNode(nonfan);
// ExEnd:5
```

## الخطوة 5: حفظ المشهد كملف OBJ

أخيرًا، **نحفظ المشهد كملف OBJ** حتى يمكنك عرض النتيجة في أي عارض ثلاثي الأبعاد قياسي.

```java
// ExStart:6
// Save scene
scene.save("Your Document Directory" + "CreateFanCylinder.obj", FileFormat.WAVEFRONTOBJ);
// ExEnd:6
```

تهانينا! لقد نجحت في **إضافة عقدة فرعية**، وتحديد موضع الكائنات، وتصدير النموذج.

## المشكلات الشائعة والنصائح

| المشكلة | الحل |
|-------|----------|
| **الملف غير موجود** عند الحفظ | تأكد من وجود المجلد الهدف وأن لديك صلاحيات الكتابة. |
| **الأسطوانة تظهر مسطحة** | تحقق من قيم نصف القطر والارتفاع؛ Aspose.3D يتوقع الوحدات بنفس المقياس. |
| **شريحة المروحة غير مكتملة** | عدّل `ThetaLength` (بالراديان) لتغطية الزاوية المطلوبة. |
| **المشهد غير مرئي في العارض** | تأكد من حفظ ملف OBJ مع ملف `.mtl` (المواد) المصاحب إذا لزم الأمر. |
