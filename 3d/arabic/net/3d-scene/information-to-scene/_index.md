# كيفية إضافة البيانات الوصفية: استخراج المعلومات إلى أصول المشهد

## المقدمة

في هذا الدرس الشامل ستكتشف **كيفية إضافة البيانات الوصفية** إلى مشاهدك ثلاثية الأبعاد باستخدام Aspose.3D for .NET. إضافة البيانات الوصفية مثل تفاصيل البائع، وحدات القياس المخصصة، ومعلومات الأصول الأخرى تجعل نماذجك أكثر معلوماتية، قابلة للبحث، وجاهزة لسلاسل المعالجة اللاحقة مثل محركات الألعاب أو منصات AR/VR.

## إجابات سريعة
- **ما هو الهدف الأساسي؟** تضمين البيانات الوصفية (معلومات البائع، الوحدات، العلامات المخصصة) مباشرةً في مشهد ثلاثي الأبعاد.  
- **ما المكتبة المستخدمة؟** Aspose.3D for .NET.  
- **هل يمكنني تصدير النموذج بعد إضافة البيانات الوصفية؟** نعم – يمكنك **تصدير نموذج ثلاثي الأبعاد**، على سبيل المثال حفظ كملف FBX.  
- **هل أحتاج إلى ترخيص؟** تتوفر نسخة تجريبية مجانية؛ يلزم الحصول على ترخيص تجاري للإنتاج.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+، .NET Core 3.1+، .NET 5/6.

## ما هو “كيفية إضافة البيانات الوصفية” في مشهد ثلاثي الأبعاد؟

إضافة البيانات الوصفية تعني إرفاق معلومات وصفية—مثل اسم التطبيق، البائع، وحدات القياس، أو العلامات المخصصة—بملف المشهد نفسه. هذه البيانات تنتقل مع النموذج عندما تقوم **بحفظ المشهد كملف FBX** أو صيغ أخرى مدعومة، مما يمكّن الأدوات اللاحقة من فهم السياق دون الحاجة إلى ملفات خارجية.

## لماذا يتم تضمين البيانات الوصفية المخصصة ومعلومات البائع؟

- **قابلية البحث:** يمكن لأنظمة إدارة الأصول تصفية النماذج حسب البائع أو نوع الوحدة.  
- **التشغيل البيني:** يمكن للمحركات التي تقرأ ملفات FBX تطبيق المقياس الصحيح تلقائيًا إذا قمت **بتعريف وحدات القياس**.  
- **العلامة التجارية:** تضمين اسم التطبيق والبائع يعزز الملكية والامتثال للترخيص.

## استيراد المساحات الاسمية

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
```

## الخطوة 1: تهيئة مشهد ثلاثي الأبعاد

```csharp
Scene scene = new Scene();
```

إنشاء كائن `Scene` جديد سيحمل جميع المعلومات الهندسية ومعلومات الأصول.

## الخطوة 2: تعيين التطبيق و **إضافة معلومات البائع**

```csharp
scene.AssetInfo.ApplicationName = "Egypt";
scene.AssetInfo.ApplicationVendor = "Manualdesk";
```

هنا نقوم بتضمين **اسم التطبيق** و **معلومات البائع**. هذا جزء أساسي من **كيفية إضافة البيانات الوصفية** التي تحدد مصدر النموذج.

## الخطوة 3: **تعريف وحدات القياس** لتدرج دقيق

```csharp
scene.AssetInfo.UnitName = "pole";
scene.AssetInfo.UnitScaleFactor = 0.6;
```

من خلال تحديد `UnitName` و `UnitScaleFactor`، تخبر الأدوات اللاحقة أن “pole” واحدة تساوي 0.6 متر (60 سم). هذه الخطوة ضرورية عندما تقوم لاحقًا **بتصدير نموذج ثلاثي الأبعاد** لضمان أبعاد واقعية صحيحة.

## الخطوة 4: **حفظ المشهد كملف FBX** – **تصدير نموذج ثلاثي الأبعاد** مع البيانات الوصفية

```csharp
var output = "Your Output Directory" + "InformationToScene.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

نداء `Save` يكتب المشهد إلى ملف FBX، مدمجًا جميع البيانات الوصفية التي أضفناها. هذا يوضح **كيفية إضافة البيانات الوصفية** ثم **حفظ المشهد كملف FBX**، وبالتالي **تصدير نموذج ثلاثي الأبعاد** مع جميع معلومات الأصول.

## الخطوة 5: عرض رسالة النجاح

```csharp
Console.WriteLine("\nAsset information added successfully to Scene.\nFile saved at " + output);
```

رسالة وحدة التحكم الودية تؤكد أن البيانات الوصفية قد تم كتابتها وموقع الملف.

## المشكلات الشائعة والنصائح

- **مقياس الوحدة غير صحيح:** تحقق مرة أخرى من أن `UnitScaleFactor` يطابق التحويل الواقعي؛ وإلا قد تظهر النماذج المصدرة كبيرة جدًا أو صغيرة جدًا.  
- **معلومات البائع مفقودة:** إذا تُرك `ApplicationVendor` فارغًا، قد يعرض بعض العارضين “Unknown”. احرص دائمًا على تعيينه عندما يكون ممكنًا.  
- **أخطاء مسار الملف:** تأكد من وجود دليل الإخراج؛ وإلا سيؤدي `scene.Save` إلى رمي استثناء.
