# كيفية إضافة معلومات البائع وحفظ مشهد FBX باستخدام Aspose.3D

## مقدمة

مرحبًا بكم في هذا الدرس الشامل الذي يوضح **كيفية إضافة معلومات البائع** إلى مشهد ثلاثي الأبعاد ثم **كيفية حفظ ملفات FBX** باستخدام Aspose.3D لـ .NET. إما كنتم تبنون تصورًا معماريًا، أو أصول ألعاب، أو نماذج هندسية، وإدراج بيانات الناشرين وتطبيقات الميتاداتا تجعل المشاهدة أكثر إيضاحًا وأسهل في الإدارة لاحقًا. دعونا نتبع عمليةًً.

## إجابات سريعة
- **ماذا يعني “إضافة بائع”?**يقوم بتخزين أسماء التطبيق داخل كتلة AssetInfo الخاصة بالمشهد.
- **أي تنسيق معتمد لمعلومات البائع؟**يحتفظ بـ FBX (ASCII أو ثنائي) بالميتاداتا عند الحفظ.
- **كيف يتم حفظ FBX؟**استخدم `scene.Save(path, FileFormat.FBX7500ASCII)` أو ما براه من صيغة ثنائية.
- **هل تحتاج إلى ترخيص؟**الإصدار المجاني المجاني للطوير؛ يتطلب مذكرة رسمية للإنتاج.
- **هل يمكنني تغيير وحدات القياس؟**نعم، قم بعمل `AssetInfo.UnitName` و `AssetInfo.UnitScaleFactor`.

## ما هي "كيفية إضافة بائع" في مشهد ثلاثي الأبعاد؟

إضافة معلومات البائع تعني إنشاء خصائص `AssetInfo` لكائن `Scene`. هذه المزرعة تأتي مع الملف، الذي يسمح للمستهلك بملف FBX برؤية أي تطبيق جاهز وهو البائع.

## لماذا تضيف معلومات البائع؟
- **إمكانية التتبع:** تحديد نموذج المصدر بسرعة في خطوط الإنتاج الكبيرة.
- **الاساس:** يتطلب بعض الاستخدامات علامة تجارية أصلية للتكنولوجيات الأصلية.
- **المحاسبة:** يمكن للسكربتات تصفية أو رفع الملفات على ميتاداتا البائع.

## المتطلبات الأساسية

- تم تثبيت Aspose.3D لـ .NET. يمكنك تنزيله من [صفحة Aspose.3D لـ .NET](https://releases.aspose.com/3d/net/).

## استيراد مساحات الأسماء

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
```

## كيفية إضافة معلومات المورّد

### الخطوة 1: تهيئة مشهد ثلاثي الأبعاد

```csharp
Scene scene = new Scene();
```

إنشاء `Scene` جديد يمنحك لوحة رسم نظيفة للعمل عليها.

### الخطوة 2: تعيين معلومات التطبيق والمورّد

```csharp
scene.AssetInfo.ApplicationName = "Egypt";
scene.AssetInfo.ApplicationVendor = "Manualdesk";
```

هنا نوضح **كيفية إضافة بيانات البائع** عن طريق تعيين سلاسل نصية ذات معنى إلى `ApplicationName` و `ApplicationVendor`.

### الخطوة 3: تحديد وحدات القياس

```csharp
scene.AssetInfo.UnitName = "pole";
scene.AssetInfo.UnitScaleFactor = 0.6;
```

تحديد نظام الوحدات يضمن أن أي شخص يفتح ملف FBX سيفهم الأبعاد بشكل صحيح. في هذا المثال، يساوي “pole” واحد 60 سم.

## كيفية حفظ مشهد FBX

### الخطوة 4: حفظ المشهد (كيفية حفظ ملف FBX)

```csharp
var output = "Your Output Directory" + "InformationToScene.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

هذا السطر يوضح **كيفية حفظ fbx** باستخدام نسخة ASCII من FBX 7.5.0. إذا كنت تفضل الصيغة الثنائية، استبدل `FBX7500ASCII` بـ `FBX7500Binary`.

> **نصيحة احترافية:** حافظ على امتداد الملف `.fbx` متوافقًا مع الصيغة التي تختارها؛ وإلا قد يفسر بعض العارضين المحتوى بشكل خاطئ.

### الخطوة 5: عرض رسالة النجاح

```csharp
Console.WriteLine("\nAsset information added successfully to Scene.\nFile saved at " + output);
```

رسالة صديقة في وحدة التحكم تؤكد أن المشهد، مكتمل بميتاداتا البائع، قد تم كتابته إلى القرص.

## المشكلات والحلول الشائعة

| مشكلة | الحل |
|-------|----------|
| **معلومات البائع غير ظاهرة في العارض** | تأكد من حفظ الملف كـ **FBX ASCII** أو **Binary**؛ بعض العارضين الأصليين يقرؤون صيغة واحدة فقط. |
| **المسار يحتوي على مسافات** | ضع المسار بين علامات المرسل أو استخدم `Path.Combine` لإنشاء مسار ملف آمن. |
| **مقياس الوحدة خاطئًا** | تحقق مرة أخرى من `UnitScaleFactor`؛ هو مضاعف لـ وحيدة. |
| **استثناء رمضان** | استخدم النسخة التجريبية للاختبار؛ واحصل على ترخيص كامل لإنشاءات الإنتاج. |
