# كيفية تحريك المكعب في مشاهد ثلاثية الأبعاد باستخدام Aspose.3D لـ .NET

## مقدمة

إذا كنت تغوص في عالم إنشاء المشاهد ثلاثية الأبعاد وتحريكها في .NET، فإن Aspose.3D هو مجموعة الأدوات المفضلة لديك. في هذا الدليل خطوة بخطوة، سنستكشف **كيفية تحريك المكعب** عن طريق تحريك خصائصه، وإنشاء منحنيات التحريك، وربط الإطارات المفتاحية. في النهاية، ستحصل على مكعب متحرك بالكامل يمكنك تصديره إلى صيغ ثلاثية الأبعاد شائعة.

## إجابات سريعة
- **ما المكتبة المستخدمة؟** Aspose.3D for .NET  
- **ما المهمة الأساسية التي يغطيها هذا الدرس؟** كيفية تحريك المكعب في مشهد ثلاثي الأبعاد  
- **خطوات رئيسية؟** استيراد المساحات الاسمية، إنشاء مشهد، ربط الإطارات المفتاحية، حفظ الملف  
- **هل أحتاج إلى ترخيص؟** النسخة التجريبية المجانية تكفي للتعلم؛ يلزم ترخيص تجاري للإنتاج  
- **صيغة الإخراج المدعومة؟** FBX (ASCII 7500) وغيرها المدعومة من قبل Aspose.3D  

## ما هو “كيفية تحريك المكعب” في Aspose.3D؟
تحريك المكعب يعني تعديل خصائص التحويل الخاصة به (مثل Translation أو Rotation أو Scale) مع مرور الوقت باستخدام بيانات الإطارات المفتاحية. توفر Aspose.3D واجهة برمجة تطبيقات نظيفة لإنشاء **منحنيات التحريك**، **ربط الإطارات المفتاحية**، و**تحريك خصائص ثلاثية الأبعاد** مباشرة على عقد المشهد.

## لماذا تحريك مكعب باستخدام Aspose.3D؟
- **تكامل كامل مع .NET** – يعمل مع .NET Framework و .NET Core و .NET 5/6.  
- **لا توجد تبعيات خارجية** – لا حاجة لـ Unity أو محركات أخرى للرسوم المتحركة البسيطة.  
- **مرونة التصدير** – يمكن حفظ المشاهد المتحركة كـ FBX أو OBJ أو GLTF، إلخ، لخطوط الأنابيب اللاحقة.

## المتطلبات المسبقة

قبل أن نبدأ هذه الرحلة المثيرة، تأكد من توفر المتطلبات التالية:

- Aspose.3D for .NET: تأكد من تثبيت المكتبة. يمكنك تنزيلها من [موقع Aspose.3D](https://releases.aspose.com/3d/net/).
- المعرفة بـ C#: الإلمام بلغة البرمجة C# ضروري لفهم وتنفيذ الأمثلة.
- بيئة التطوير المتكاملة (IDE): استخدم IDE المفضلة لديك، مثل Visual Studio، للبرمجة مع الأمثلة.
- مفاهيم أساسية للمشهد ثلاثي الأبعاد: فهم المفاهيم الأساسية للمشهد سيسهل رحلتك التعليمية.

## استيراد المساحات الاسمية

في كود C# الخاص بك، تأكد من استيراد المساحات الاسمية الضرورية لـ Aspose.3D. إليك المجموعة المطلوبة:

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
using Aspose.ThreeD.Animation;
using Aspose.ThreeD.Entities;
using Aspose.ThreeD.Utilities;
using Aspose._3D.Examples.CSharp.Geometry_Hierarchy;
```

## الخطوة 1: تهيئة كائن المشهد

إنشاء مشهد فارغ سيحتوي على جميع العقد والرسوم المتحركة.

```csharp
Scene scene = new Scene();
```

## الخطوة 2: إنشاء شبكة باستخدام Polygon Builder

نقوم بإنشاء شبكة مكعب بسيطة باستخدام الطريقة المساعدة `Common.CreateMeshUsingPolygonBuilder()`.

```csharp
Mesh mesh = Common.CreateMeshUsingPolygonBuilder();
```

## الخطوة 3: إنشاء عقد المكعب

أضف شبكة المكعب إلى المشهد كعقدة فرعية للجذر. سيتم استخدام اسم العقدة **cube1** لاحقًا عند ربط الإطارات المفتاحية.

```csharp
Node cube1 = scene.RootNode.CreateChildNode("cube1", mesh);
```

## الخطوة 4: العثور على خاصية Translation

لتحريك موقع المكعب، نحتاج إلى تحديد خاصية **Translation** على تحويل العقدة.

```csharp
Property translation = cube1.Transform.FindProperty("Translation");
```

## الخطوة 5: إنشاء Bind Point

`BindPoint` يربط خاصية في المشهد بمنحنى تحريك. هنا نربط خاصية الترجمة.

```csharp
BindPoint bindPoint = new BindPoint(scene, translation);
```

## الخطوة 6: ربط منحنى التحريك على المكوّن X

نقوم الآن بإنشاء منحنى تحريك للمحور **X**. هذا يوضح خطوة **إنشاء منحنى التحريك** ويظهر كيفية **ربط الإطارات المفتاحية**.

```csharp
bindPoint.BindKeyframeSequence("X", new KeyframeSequence()
{
    {0, 10.0f, Interpolation.Bezier},
    {3, 20.0f, Interpolation.Bezier},
    {5, 30.0f, Interpolation.Linear},
});
```

## الخطوة 7: ربط منحنى التحريك على المكوّن Z

وبالمثل، نحرك المحور **Z** لإعطاء المكعب مسار حركة أكثر ديناميكية.

```csharp
bindPoint.BindKeyframeSequence("Z", new KeyframeSequence()
{
    {0, 10.0f, Interpolation.Bezier},
    {3, -10.0f, Interpolation.Bezier},
    {5, 0.0f, Interpolation.Linear},
});
```

## الخطوة 8: حفظ المشهد ثلاثي الأبعاد

تصدير المشهد المتحرك إلى ملف FBX. الصيغة `FBX7500ASCII` مدعومة على نطاق واسع من قبل أدوات 3D.

```csharp
string output = "Your Output Directory" + "PropertyToDocument.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

## الخطوة 9: عرض رسالة النجاح

إعطاء المستخدم ملاحظة بأن التحريك تم إضافته بنجاح.

```csharp
Console.WriteLine("\nAnimation property added successfully to document.\nFile saved at " + output);
```

## تصدير المشهد المتحرك إلى FBX

أحد أكثر المهام شيوعًا بعد تحريك المكعب هو **تصدير المشهد المتحرك FBX** حتى تتمكن التطبيقات ثلاثية الأبعاد الأخرى من استهلاكه. الكود أعلاه بالفعل يحفظ المشهد بصيغة FBX7500ASCII، التي تحافظ على بيانات الإطارات المفتاحية وتعمل بسلاسة مع أدوات مثل Autodesk Maya و Blender و Unity. عند فتح ملف `.fbx` الناتج، يجب أن ترى المكعب يتحرك على محوري X و Z تمامًا كما تم تعريفه في تسلسلات الإطارات المفتاحية.

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|-------|--------|-----|
| عدم ملاحظة الحركة | أوقات الإطارات المفتاحية لا تتطابق مع نطاق التشغيل | تأكد من أن جدول زمني للرسوم المتحركة في المشهد يغطي أوقات الإطارات المفتاحية (0‑5 ثوانٍ في هذا المثال). |
| مسار ملف غير صحيح | `output` يشير إلى دليل غير موجود | أنشئ الدليل أولاً أو استخدم مسارًا مطلقًا. |
| استثناء الترخيص | التشغيل بدون ترخيص صالح في الإنتاج | طبق ترخيص Aspose.3D قبل إنشاء الـ `Scene`. |

## أسئلة إضافية (محسّنة بالذكاء الاصطناعي)

**س: هل يمكنني تحريك خصائص أخرى مثل الدوران أو المقياس؟**  
ج: بالتأكيد. استخدم `cube1.Transform.FindProperty("Rotation")` أو `"Scale"` وربط تسلسلات الإطارات المفتاحية بنفس الطريقة.

**س: هل يدعم Aspose.3D أنواع تقاطع الإطارات المفتاحية غير Bezier و Linear؟**  
ج: نعم، يدعم أيضًا `Interpolation.Step` و `Interpolation.Cubic` لمزيد من التحكم.

**س: كيف يمكنني معاينة التحريك قبل التصدير؟**  
ج: يوفر Aspose.3D عارضًا بسيطًا في API؛ بدلاً من ذلك، يمكنك تحميل ملف FBX المصدر في عارض ثلاثي الأبعاد مثل Autodesk FBX Review.

**س: هل من الممكن تحريك عدة مكعبات في وقت واحد؟**  
ج: أنشئ عقدًا منفصلة لكل مكعب، احصل على خصائصها الخاصة، وربط تسلسلات إطارات مفتاحية مستقلة.
