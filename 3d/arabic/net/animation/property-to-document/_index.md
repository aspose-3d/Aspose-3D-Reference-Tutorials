# كيفية تحريك المكعب في مشاهد 3D باستخدام Aspose.3D لـ .NET

## المقدمة

إذا كنت تستكشف عالم إنشاء المشاهد الثلاثية الأبعاد والتحريك في .NET، فإن Aspose.3D هو مجموعة الأدوات التي تعتمد عليها. في هذا الدليل خطوة بخطوة، سنستعرض **كيفية تحريك مكعب** عن طريق تحريك خصائصه، إنشاء منحنيات التحريك، وربط الإطارات المفتاحية. في النهاية، ستحصل على مكعب متحرك بالكامل يمكنك تصديره إلى صيغ 3D شائعة.

## إجابات سريعة
- **ما المكتبة المستخدمة؟** Aspose.3D لـ .NET  
- **ما المهمة الأساسية التي يغطيها هذا الدرس؟** كيفية تحريك مكعب في مشهد 3D  
- **ما الخطوات الرئيسية؟** استيراد المساحات الاسمية، إنشاء مشهد، ربط الإطارات المفتاحية، حفظ الملف  
- **هل أحتاج إلى ترخيص؟** نسخة تجريبية مجانية تكفي للتعلم؛ الترخيص التجاري مطلوب للإنتاج  
- **ما صيغ الإخراج المدعومة؟** FBX (ASCII 7500) وغيرها التي يدعمها Aspose.3D  

## ما هو “كيفية تحريك مكعب” في Aspose.3D؟
تحريك مكعب يعني تعديل خصائص التحويل الخاصة به (مثل الإزاحة، الدوران، أو المقياس) مع مرور الوقت باستخدام بيانات الإطارات المفتاحية. توفر Aspose.3D واجهة برمجة تطبيقات نظيفة لإنشاء **منحنيات التحريك**، **ربط الإطارات المفتاحية**، و**تحريك الخصائص ثلاثية الأبعاد** مباشرة على عقد المشهد.

## لماذا نتحرك مكعبًا باستخدام Aspose.3D؟
- **تكامل كامل مع .NET** – يعمل مع .NET Framework، .NET Core، و .NET 5/6.  
- **بدون تبعيات خارجية** – لا حاجة إلى Unity أو محركات أخرى للرسوم المتحركة البسيطة.  
- **مرونة التصدير** – يمكن حفظ المشاهد المتحركة كـ FBX، OBJ، GLTF، إلخ، لتدفقات العمل اللاحقة.

## المتطلبات المسبقة

قبل أن نبدأ هذه الرحلة المثيرة، تأكد من توفر المتطلبات التالية:

- Aspose.3D لـ .NET: تأكد من تثبيت المكتبة. يمكنك تنزيلها من [موقع Aspose.3D](https://releases.aspose.com/3d/net/).

- معرفة بلغة C#: الإلمام بلغة البرمجة C# ضروري لفهم وتنفيذ الأمثلة.

- بيئة تطوير متكاملة (IDE): استخدم IDE المفضلة لديك، مثل Visual Studio، للبرمجة مع الأمثلة.

- مفاهيم أساسية للمشهد ثلاثي الأبعاد: فهم أساسي لمفاهيم المشهد ثلاثي الأبعاد سيسهل عملية التعلم.

## استيراد المساحات الاسمية

في كود C# الخاص بك، تأكد من استيراد المساحات الاسمية اللازمة لـ Aspose.3D. إليك المجموعة المطلوبة:

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
using Aspose.ThreeD.Animation;
using Aspose.ThreeD.Entities;
using Aspose.ThreeD.Utilities;
using Aspose._3D.Examples.CSharp.Geometry_Hierarchy;
```

## الخطوة 1: تهيئة كائن المشهد

إنشاء مشهد فارغ سيحمل جميع العقد والتحريكات الخاصة بنا.

```csharp
Scene scene = new Scene();
```

## الخطوة 2: إنشاء شبكة باستخدام Polygon Builder

نولد شبكة مكعب بسيطة باستخدام الطريقة المساعدة `Common.CreateMeshUsingPolygonBuilder()`.

```csharp
Mesh mesh = Common.CreateMeshUsingPolygonBuilder();
```

## الخطوة 3: إنشاء عقد المكعب

أضف شبكة المكعب إلى المشهد كعقدة فرعية للجذر. سيتم استخدام اسم العقدة **cube1** لاحقًا عند ربط الإطارات المفتاحية.

```csharp
Node cube1 = scene.RootNode.CreateChildNode("cube1", mesh);
```

## الخطوة 4: العثور على خاصية الإزاحة (Translation)

لتحريك موقع المكعب، نحتاج إلى تحديد خاصية **Translation** على تحويل العقدة.

```csharp
Property translation = cube1.Transform.FindProperty("Translation");
```

## الخطوة 5: إنشاء نقطة ربط (Bind Point)

`BindPoint` يربط خاصية المشهد بمنحنى التحريك. هنا نربط خاصية الإزاحة.

```csharp
BindPoint bindPoint = new BindPoint(scene, translation);
```

## الخطوة 6: ربط منحنى التحريك للمكوّن X

ننشئ الآن منحنى تحريك للمحور **X**. يوضح هذا خطوة **إنشاء منحنى التحريك** ويظهر كيفية **ربط الإطارات المفتاحية**.

```csharp
bindPoint.BindKeyframeSequence("X", new KeyframeSequence()
{
    {0, 10.0f, Interpolation.Bezier},
    {3, 20.0f, Interpolation.Bezier},
    {5, 30.0f, Interpolation.Linear},
});
```

## الخطوة 7: ربط منحنى التحريك للمكوّن Z

بنفس الطريقة، نحرك المحور **Z** لإعطاء المكعب مسار حركة أكثر ديناميكية.

```csharp
bindPoint.BindKeyframeSequence("Z", new KeyframeSequence()
{
    {0, 10.0f, Interpolation.Bezier},
    {3, -10.0f, Interpolation.Bezier},
    {5, 0.0f, Interpolation.Linear},
});
```

## الخطوة 8: حفظ المشهد ثلاثي الأبعاد

تصدير المشهد المتحرك إلى ملف FBX. الصيغة `FBX7500ASCII` مدعومة على نطاق واسع من أدوات 3D.

```csharp
string output = "Your Output Directory" + "PropertyToDocument.fbx";
scene.Save(output, FileFormat.FBX7500ASCII);
```

## الخطوة 9: عرض رسالة النجاح

إعطاء المستخدم تغذية راجعة بأن التحريك أضيف بنجاح.

```csharp
Console.WriteLine("\nAnimation property added successfully to document.\nFile saved at " + output);
```

## المشكلات الشائعة والحلول

| المشكلة | السبب | الحل |
|--------|-------|------|
| لا يُلاحظ أي حركة | أوقات الإطارات المفتاحية لا تتطابق مع نطاق التشغيل | تأكد من أن خط الزمن للمشهد يغطي أوقات الإطارات المفتاحية (0‑5 ثوانٍ في هذا المثال). |
| مسار الملف غير صحيح | المتغيّر `output` يشير إلى دليل غير موجود | أنشئ الدليل أولاً أو استخدم مسارًا مطلقًا. |
| استثناء الترخيص | تشغيل بدون ترخيص صالح في بيئة الإنتاج | طبّق ترخيص Aspose.3D قبل إنشاء كائن `Scene`. |

## أسئلة إضافية (محسّنة بالذكاء الاصطناعي)

**س: هل يمكنني تحريك خصائص أخرى مثل الدوران أو المقياس؟**  
ج: بالتأكيد. استخدم `cube1.Transform.FindProperty("Rotation")` أو `"Scale"` وربط تسلسلات الإطارات المفتاحية بنفس الطريقة.

**س: هل يدعم Aspose.3D أنواع تقاطع الإطارات المفتاحية غير Bezier و Linear؟**  
ج: نعم، يدعم أيضًا `Interpolation.Step` و `Interpolation.Cubic` لمزيد من التحكم.

**س: كيف يمكنني معاينة التحريك قبل التصدير؟**  
ج: يوفر Aspose.3D عارضًا بسيطًا في API؛ بدلاً من ذلك، حمّل ملف FBX المصدر في عارض ثلاثي الأبعاد مثل Autodesk FBX Review.

**س: هل يمكن تحريك عدة مكعبات في آنٍ واحد؟**  
ج: أنشئ عقدًا منفصلة لكل مكعب، استخرج خصائصها الخاصة، وربط تسلسلات إطارات مفتاحية مستقلة.
