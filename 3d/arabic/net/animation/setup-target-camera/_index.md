# إضافة كاميرا إلى المشهد وإعداد الأهداف للرسوم المتحركة ثلاثية الأبعاد

## مقدمة

إذا كنت تبني رسماً متحركاً ثلاثي الأبعاد، فإن أول شيء تحتاجه هو كاميرا موضوعة بشكل جيد. في هذا الدرس ستتعلم **كيفية إضافة كاميرا إلى المشهد**، تعريف هدفها، وأخيرًا **تصدير المشهد كملف FBX** باستخدام Aspose.3D for .NET. سنستعرض كل خطوة، نشرح سبب أهميتها، ونقدم لك نصائح عملية حتى تتمكن من إنشاء رسومات متحركة ثلاثية الأبعاد جذابة بثقة. في النهاية ستفهم أيضًا **كيفية وضع الكاميرا في الفضاء ثلاثي الأبعاد** للحصول على تأطير مثالي.

## إجابات سريعة
- **ما هي الخطوة الأولى لإضافة كاميرا؟** تهيئة كائن `Scene` الذي سيستضيف عقدة الكاميرا.  
- **ما هو تنسيق الملف المستخدم للتصدير في هذا الدليل؟** FBX (`.fbx`).  
- **هل أحتاج إلى ترخيص لتشغيل كود العينة؟** النسخة التجريبية المجانية تكفي للتقييم؛ الترخيص التجاري مطلوب للإنتاج.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+، .NET Core 3.1+، .NET 5/6+.  
- **هل يمكنني تغيير موضع الكاميرا بعد إنشائها؟** نعم – عدل `cameraNode.Transform.Translation` في أي وقت.

## ما هو **add camera to scene**؟
إضافة كاميرا إلى المشهد تعني إنشاء كيان `Camera`، ربطه بعقدة في رسم المشهد، وتحديد موضعه بحيث “ينظر إلى” نقطة هدف. هذا يحدد منظور المشاهد عند عرض المشهد أو تصديره.

## لماذا إعداد أهداف الكاميرا وتصديرها كـ FBX؟
- **التحكم في وجهة النظر** – وضع الكاميرا بدقة يتيح لك تأطير الرسوم المتحركة كما تتخيل.  
- **قابلية التبادل** – FBX مدعوم على نطاق واسع من قبل محركات الألعاب، Maya، Blender، وغيرها من أدوات 3D، مما يسهل مشاركة الأصول.  
- **أصول قابلة لإعادة الاستخدام** – بمجرد حفظ الكاميرا وهدفها، يمكنك إعادة استخدام المشهد في مشاريع متعددة.

## حالات الاستخدام الشائعة
- **مشاهد سينمائية** في الألعاب حيث تتبع كاميرا ثابتة شخصية.  
- **تصوير منتجات** حيث تحتاج إلى منظور ثابت لعرض النموذج من زوايا مختلفة.  
- **معاينة مسبقة** لأفلام أو مشاريع AR/VR، مما يسمح للمصممين بتجربة موضع الكاميرا قبل الإخراج النهائي.

## المتطلبات المسبقة

قبل الغوص في الدرس، تأكد من توفر المتطلبات التالية:

- معرفة أساسية بـ C# وإطار عمل .NET.  
- مكتبة Aspose.3D for .NET مثبتة. يمكنك تنزيلها [هنا](https://releases.aspose.com/3d/net/).  
- بيئة تطوير جاهزة للبرمجة ثلاثية الأبعاد.

## استيراد مساحات الأسماء

لبدء العملية، استورد مساحات الأسماء الضرورية في مشروعك. هذه المساحات أساسية للاستفادة من قوة Aspose.3D for .NET:

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
using Aspose.ThreeD.Animation;
using Aspose.ThreeD.Entities;
using Aspose.ThreeD.Utilities;
```

## دليل خطوة بخطوة

### الخطوة 1: تهيئة كائن المشهد

ابدأ بتهيئة كائن المشهد. هذا يعمل كقماش ستُظهر فيه رسومتك المتحركة ثلاثية الأبعاد.

```csharp
// ExStart:SetupTargetAndCamera
// Initialize scene object
Scene scene = new Scene();
```

### الخطوة 2: إنشاء عقدة كاميرا

بعد ذلك، أنشئ عقدة فرعية ستحمل الكاميرا. إعطاء العقدة اسمًا معبرًا يساعد في تنظيم هيكلية المشهد.

```csharp
// Get a child node object
Node cameraNode = scene.RootNode.CreateChildNode("camera", new Camera());
```

### الخطوة 3: وضع الكاميرا

حدد إزاحة (translation) عقدة الكاميرا. هذا يحدد الموضع الأولي للكاميرا في الفضاء ثلاثي الأبعاد. عدل قيم `Vector3` **لوضع الكاميرا في الفضاء ثلاثي الأبعاد** حسب الحاجة للقطتك.

```csharp
// Set camera node translation
cameraNode.Transform.Translation = new Vector3(100, 20, 0);
```

### الخطوة 4: تعريف هدف الكاميرا

تحتاج الكاميرا إلى نقطة هدف لتنظر إليها. ننشئ عقدة فرعية أخرى تعمل كنقطة تركيز ونعيّنها لخاصية `Target` الخاصة بالكاميرا. هذه هي الطريقة التي **تضبط بها هدف الكاميرا** للحصول على عرض ثابت.

```csharp
cameraNode.GetEntity<Camera>().Target = scene.RootNode.CreateChildNode("target");
```

### الخطوة 5: حفظ المشهد المُكوَّن

أخيرًا، صدّر المشهد إلى ملف FBX (أو أي تنسيق مدعوم آخر). استبدل `"Your Output Directory"` بالمسار الفعلي حيث تريد حفظ الملف.

```csharp
var output = "Your Output Directory" + "camera-test.fbx";
scene.Save(output);
```

## المشكلات الشائعة والحلول

| المشكلة | الحل |
|-------|----------|
| **تظهر الكاميرا بزاوية خاطئة** | تأكد من أن عقدة الهدف موضوعة في الموضع المتوقع. يمكنك أيضًا ضبط `cameraNode.GetEntity<Camera>().UpVector` للتحكم في الاتجاه. |
| **ملف FBX المُصدّر لا يفتح في أدوات أخرى** | تأكد من أنك تستخدم نسخة حديثة من Aspose.3D (المكتبة تكتب تلقائيًا نسخة FBX متوافقة). |
| **خطأ “المسار غير موجود” عند `scene.Save`** | استخدم مسارًا مطلقًا أو تأكد من وجود دليل الإخراج قبل استدعاء `Save`. |
