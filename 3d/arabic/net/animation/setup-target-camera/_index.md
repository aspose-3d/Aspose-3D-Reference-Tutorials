# إضافة كاميرا إلى المشهد وإعداد الأهداف للرسوم المتحركة ثلاثية الأبعاد

## مقدمة

## إجابات سريعة
- **ما هي الخطوة الأولى لإضافة كاميرا؟** تهيئة كائن `Scene` الذي سيستضيف عقدة الكاميرا.  
- **ما هو تنسيق الملف المستخدم للتصدير في هذا الدليل؟** FBX (`.fbx`).  
- **هل أحتاج إلى ترخيص لتشغيل عينة الكود؟** النسخة التجريبية المجانية تكفي للتقييم؛ الترخيص التجاري مطلوب للإنتاج.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.5+، .NET Core 3.1+، .NET 5/6+.  
- **هل يمكنني تغيير موقع الكاميرا بعد الإنشاء؟** نعم – يمكن تعديل `cameraNode.Transform.Translation` في أي وقت.

## ما هو **إضافة كاميرا إلى المشهد**؟
إضافة كاميرا إلى المشهد تعني إنشاء كيان `Camera`، وربطه بعقدة في رسم المشهد، وتحديد موقعه بحيث “ينظر إلى” نقطة هدف. هذا يحدد منظور المشاهد عند عرض المشهد أو تصديره.

## لماذا إعداد أهداف الكاميرا وتصديرها كـ FBX؟
- **التحكم في وجهة النظر** – يتيح وضع الكاميرا بدقة تأطير الرسوم المتحركة بالضبط كما تتخيله.  
- **قابلية التبادل** – FBX مدعوم على نطاق واسع من قبل محركات الألعاب، Maya، Blender، وغيرها من أدوات 3D، مما يسهل مشاركة الأصول.  
- **أصول قابلة لإعادة الاستخدام** – بمجرد حفظ الكاميرا والهدف الخاص بها، يمكنك إعادة استخدام المشهد في مشاريع متعددة.

## المتطلبات المسبقة

قبل الغوص في الدرس، تأكد من توفر المتطلبات التالية:

- معرفة أساسية بـ C# وإطار عمل .NET.  
- مكتبة Aspose.3D لـ .NET مثبتة. يمكنك تنزيلها [هنا](https://releases.aspose.com/3d/net/).  
- بيئة تطوير جاهزة لبرمجة 3D.

## استيراد مساحات الأسماء

لبدء العملية، استورد مساحات الأسماء الضرورية في مشروعك. هذه المساحات أساسية لاستغلال قوة Aspose.3D لـ .NET:

```csharp
using System;
using System.IO;
using System.Collections;
using Aspose.ThreeD;
using Aspose.ThreeD.Animation;
using Aspose.ThreeD.Entities;
using Aspose.ThreeD.Utilities;
```

## دليل خطوة بخطوة

### الخطوة 1: تهيئة كائن المشهد

ابدأ بتهيئة كائن المشهد. هذا يعمل كقماش حيث ستتحقق الرسوم المتحركة ثلاثية الأبعاد.

```csharp
// ExStart:SetupTargetAndCamera
// Initialize scene object
Scene scene = new Scene();
```

### الخطوة 2: إنشاء عقدة كاميرا

بعد ذلك، أنشئ عقدة فرعية ستحمل الكاميرا. إعطاء العقدة اسمًا معبرًا يساعد في تنظيم هيكلية المشهد.

```csharp
// Get a child node object
Node cameraNode = scene.RootNode.CreateChildNode("camera", new Camera());
```

### الخطوة 3: وضع الكاميرا

حدد إزاحة (الترجمة) لعقدة الكاميرا. هذا يحدد الموقع الأولي للكاميرا في الفضاء ثلاثي الأبعاد.

```csharp
// Set camera node translation
cameraNode.Transform.Translation = new Vector3(100, 20, 0);
```

### الخطوة 4: تعريف هدف الكاميرا

تحتاج الكاميرا إلى نقطة هدف لتوجيه النظر إليها. نقوم بإنشاء عقدة فرعية أخرى تعمل كنقطة تركيز ونربطها بخاصية `Target` للكاميرا.

```csharp
cameraNode.GetEntity<Camera>().Target = scene.RootNode.CreateChildNode("target");
```

### الخطوة 5: حفظ المشهد المُكوَّن

أخيرًا، صدّر المشهد إلى ملف FBX (أو أي تنسيق مدعوم آخر). استبدل `"Your Output Directory"` بالمسار الفعلي حيث تريد حفظ الملف.

```csharp
var output = "Your Output Directory" + "camera-test.fbx";
scene.Save(output);
```

## المشكلات الشائعة والحلول

| المشكلة | الحل |
|-------|----------|
| **الكاميرا تظهر بزاوية خاطئة** | تحقق من أن عقدة الهدف موضوعة في المكان المتوقع. يمكنك أيضًا ضبط `cameraNode.GetEntity<Camera>().UpVector` للتحكم في الاتجاه. |
| **ملف FBX المُصدّر لا يفتح في أدوات أخرى** | تأكد من أنك تستخدم نسخة حديثة من Aspose.3D (المكتبة تكتب تلقائيًا نسخة FBX متوافقة). |
| **خطأ مسار غير موجود عند `scene.Save`** | استخدم مسارًا مطلقًا أو تأكد من وجود دليل الإخراج قبل استدعاء `Save`. |
