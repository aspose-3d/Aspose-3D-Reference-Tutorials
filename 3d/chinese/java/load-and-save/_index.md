## 常见问题与解决方案

MaterialConverter 是一个将传统材质定义转换为基于物理的渲染 (PBR) 格式的实用类。

**Issue:** *材料在转换后丢失纹理。*  
**Solution:** 在保存前使用 `MaterialConverter.convertToPbr()`；它会保留纹理贴图并自动更新着色器参数。

LoadOptions 是一个配置类，用于控制 Aspose.3D 解析输入文件的方式。

**Issue:** *大型 OBJ 文件导致 OutOfMemoryError。*  
**Solution:** 通过 `LoadOptions.setEnableMemoryOptimization(true)` 启用流式处理，并增大 JVM 堆大小（`-Xmx4g`）。

SaveOptions 配置保存场景时的输出设置，例如格式、压缩和功能包含等。

**Issue:** *导出的 FBX 文件缺少动画数据。*  
**Solution:** 设置 `SaveOptions.setExportAnimations(true)`，并确认源场景包含关键帧动画轨迹。

**Q:** *我可以在商业 Java 应用中使用 Aspose.3D 吗？*  
**A:** 可以。生产环境需要有效的 Aspose 许可证；评估可使用免费试用。

**Q:** *支持哪些 3D 文件格式的加载和保存？*  
**A:** Aspose.3D 支持 OBJ、STL、FBX、3DS、glTF、PLY、U3D、X 等多种格式，总计超过 30 种。

**Q:** *如何在不重写整个场景的情况下将材质升级为 PBR？*  
**A:** 使用 API 中提供的 `MaterialConverter.convertToPbr()` 方法——一次调用即可更新所有材质。

**Q:** *保存大型场景时有没有办法减小文件大小？*  
**A:** 有。配置 `SaveOptions`（例如启用压缩、降低网格精度）即可优化输出大小。

**Q:** *使用 Aspose.3D 是否需要额外的依赖？*
