## 常见问题

**Q: 我可以使用 Aspose.3D 为 Unity 等实时引擎创建 UV 映射吗？**  
A: 可以。将带 UV 的网格导出为 Unity 支持的格式（如 FBX 或 glTF），然后直接导入。

**Q: 三角形转换会影响原始模型的层级结构吗？**  
A: 转换会生成一个包含三角形的新网格，同时保留原始节点层级，因而变换保持不变。

**Q: 如果我的模型已经包含 UV 信息怎么办？**  
A: 只有在显式调用 UV 生成方法时，Aspose.3D 才会覆盖现有的 UV 通道；否则保持原样。

**Q: 在运行时生成 UV 会带来性能惩罚吗？**  
A: 建议在资产预处理阶段一次性生成 UV。运行时生成虽可实现，但对大型网格可能增加开销。

**Q: 哪些文件格式会保留生成的 UV 坐标？**  
A: OBJ、FBX、glTF 以及使用扩展 STL 格式的 STL 都会保存 Aspose.3D 写入的 UV 数据。
