# Geometrie und Hierarchie

## Einführung

Willkommen zum ultimativen Leitfaden für **geometric transformation 3d** mit Aspose.3D für .NET. Egal, ob Sie gerade erst anfangen oder seit Jahren 3D‑Anwendungen entwickeln, diese Sammlung praxisnaher Tutorials hilft Ihnen, alles zu meistern – vom Anwenden von PBR‑Materialien bis zum Manipulieren von Knoten mit einer Transformationsmatrix. **Sie lernen, wie man PBR anwendet – speziell, wie man pbr‑Materialien auf Ihre 3D‑Modelle anwendet**. Tauchen wir ein und sehen, wie Sie Konzepte in reale 3D‑Szenen verwandeln können.

## Schnelle Antworten
- **What is PBR?** Physically‑Based Rendering (PBR) simuliert realistische Materialreaktionen auf Licht.  
- **Why use Aspose.3D for PBR?** Es bietet eine unkomplizierte .NET‑API zur Materialerstellung und -rendering.  
- **Do I need a license?** Eine kostenlose Testversion ist verfügbar; für die Produktion ist eine kommerzielle Lizenz erforderlich.  
- **Which .NET versions are supported?** .NET Framework 4.5+, .NET Core 3.1+, .NET 5/6+.  
- **Can I combine PBR with custom shaders?** Ja, Sie können Materialien mit benutzerdefiniertem Shader‑Code erweitern.

## So wenden Sie PBR‑Materialien in Aspose.3D an
Das Anwenden von PBR‑Materialien ist das Fundament für die Erstellung fotorealistischer 3D‑Szenen. In diesem Abschnitt gehen wir die wesentlichen Schritte durch – ein Material erstellen, es der Geometrie zuweisen und das Ergebnis rendern. Der gleiche Ansatz wird in vielen der nachfolgenden Tutorials wiederverwendet, sodass Sie ihn zu beherrschen, beschleunigt jedes weitere Beispiel, das Sie erkunden.

### Warum PBR verwenden?
- **Realism:** PBR ahmt nach, wie Licht mit Oberflächen in der realen Welt interagiert.  
- **Consistency:** Materialien sehen unter jeder Beleuchtungsbedingung korrekt aus.  
- **Performance:** Optimierte Shader machen PBR für interaktive Anwendungen geeignet.

### Voraussetzungen
- Aspose.3D für .NET (neueste Version) installiert.  
- Eine .NET‑Entwicklungsumgebung (Visual Studio 2022 oder neuer).

### Wie es zu anderen Themen passt
Sobald Sie **how to apply pbr** kennen, können Sie es leicht mit anderen Transformationen wie **euler angle rotation**, Quaternion‑Mischung oder direkter Matrixmanipulation kombinieren – alles in den nachfolgenden Tutorials behandelt.

## Häufig gestellte Fragen

**Q: Wie starte ich ein neues Projekt, das PBR‑Materialien verwendet?**  
A: Installieren Sie das Aspose.3D NuGet‑Paket, erstellen Sie eine `Scene` und folgen Sie dem Tutorial „Applying PBR Material to Box“, um Ihr erstes PBR‑Material hinzuzufügen.

**Q: Kann ich diese Tutorials mit .NET Core verwenden?**  
A: Ja, alle Beispiele funktionieren mit .NET Core 3.1 und später sowie mit .NET 5/6.

**Q: Was ist der beste Weg, eine Knoten‑Hierarchie abzufragen?**  
A: Verwenden Sie die XPath‑ähnliche Abfragesyntax, die im Tutorial „XPath‑Like Object Queries“ demonstriert wird – dies ist die prägnanteste Methode, um **how to query hierarchy** zu erledigen.

**Q: Ich erhalte falsche Beleuchtung auf meinem Würfel – was sollte ich überprüfen?**  
A: Stellen Sie sicher, dass die Normalen korrekt gesetzt sind (siehe „Setting Up Normals on Cube“) und dass Ihr PBR‑Material korrekte Metall‑ und Rauheitswerte enthält.

**Q: Gibt es Leistungsaspekte bei der Verwendung vieler Transformationsmatrizen?**  
A: Stapeln Sie Transformationen, wo möglich, und wiederverwenden Sie Matrix‑Objekte, um Allokationen zu reduzieren; der Leitfaden „Transforming Node by Transformation Matrix“ enthält Best‑Practice‑Tipps.
